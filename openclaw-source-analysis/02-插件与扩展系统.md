## 一、插件体系全景

OpenClaw 有 **三层扩展机制**：

| 层级 | 名称 | 位置 | 用途 |
|------|------|------|------|
| 1 | **Extensions** | `extensions/*/` | 完整插件包（Channel/Provider/工具） |
| 2 | **Plugin SDK** | `src/plugin-sdk/` | 插件开发 SDK（供 Extension 使用） |
| 3 | **Skills** | `skills/*/` | 轻量级技能（基于文档触发） |

---

## 二、Extension 插件

### 2.1 插件清单

每个 Extension 必须包含 `openclaw.plugin.json`：

```json
{
  "id": "feishu",
  "channels": ["feishu"],
  "channelEnvVars": {
    "feishu": [
      "FEISHU_APP_ID",
      "FEISHU_APP_SECRET",
      "FEISHU_VERIFICATION_TOKEN",
      "FEISHU_ENCRYPT_KEY"
    ]
  },
  "skills": ["./skills"],
  "configSchema": {
    "type": "object",
    "properties": {}
  }
}
```

### 2.2 插件目录结构

典型的 Extension 结构：

```
extensions/feishu/
├── openclaw.plugin.json      # 插件清单（必填）
├── index.ts                  # 主入口
├── channel-entry.ts          # 通道入口（Channel 插件）
├── setup-entry.ts            # 设置入口
├── api.ts                    # API 封装
├── src/
│   ├── channel.ts            # 通道实现
│   ├── thread-bindings.ts    # 话题绑定
│   ├── outbound.ts           # 消息发送
│   └── ...
├── skills/                   # 附带 Skills
│   ├── feishu-bitable/
│   │   └── SKILL.md
│   ├── feishu-calendar/
│   │   └── SKILL.md
│   └── ...
└── test/                     # 测试
    ├── *.test.ts
    └── ...
```

### 2.3 插件类型

| 类型 | 示例 | 说明 |
|------|------|------|
| **Channel** | `feishu`, `telegram`, `discord` | 消息平台 |
| **Provider** | `openai`, `anthropic`, `google` | AI 模型提供商 |
| **Capability** | `memory-core`, `browser` | 能力扩展 |
| **Tool** | `diffs`, `lobster` | 工具集 |

## 三、Plugin SDK

### 3.1 SDK 入口

插件通过 `@openclaw/plugin-sdk` 导入 SDK：

```typescript
import { definePlugin, defineChannel, defineProvider } from "@openclaw/plugin-sdk";
```

### 3.2 SDK 子路径

SDK 按功能分拆成多个子模块：

```typescript
// 核心
import { ... } from "@openclaw/plugin-sdk";
import { ... } from "@openclaw/plugin-sdk/core";

// 路由
import { ... } from "@openclaw/plugin-sdk/routing";

// 运行时
import { ... } from "@openclaw/plugin-sdk/runtime";
import { ... } from "@openclaw/plugin-sdk/runtime-env";
import { ... } from "@openclaw/plugin-sdk/runtime-doctor";

// Provider 设置
import { ... } from "@openclaw/plugin-sdk/provider-setup";
import { ... } from "@openclaw/plugin-sdk/self-hosted-provider-setup";

// 沙箱
import { ... } from "@openclaw/plugin-sdk/sandbox";

// 密钥解析
import { ... } from "@openclaw/plugin-sdk/runtime-secret-resolution";
```

### 3.3 SDK 源码位置

```
src/plugin-sdk/
├── index.ts                  # 主入口
├── core.ts                   # 核心
├── routing.ts                # 路由
├── runtime.ts                # 运行时
├── runtime-env.ts            # 运行时环境
├── runtime-doctor.ts         # 诊断
├── provider-setup.ts         # Provider 设置
├── self-hosted-provider-setup.ts  # 自托管 Provider 设置
├── sandbox.ts                # 沙箱
├── runtime-secret-resolution.ts  # 密钥解析
└── ...
```

## 四、Skills 系统

### 4.1 Skill 结构

Skill 是**文档驱动的轻量级扩展**：

```
skills/github/
└── SKILL.md          # Skill 定义（Markdown）
```

### 4.2 SKILL.md 格式

```markdown
# GitHub Skill

<description>
GitHub 相关操作：查询仓库、提交代码、管理 Issues
</description>

<available_tools>
- name: github_query
  description: 查询 GitHub 数据
- name: github_create_issue
  description: 创建 Issue
</available_tools>

<when>
用户提到 "github"、"仓库"、"issue"、"pr" 时触发
</when>
```

## 五、插件生命周期

### 5.1 完整生命周期

```
发现 (Discovery)
    │
    ▼
加载 (Loading)
    │
    ▼
初始化 (Initialization)
    │
    ▼
激活 (Activation)
    │
    ▼
运行 (Runtime) ← 提供功能
    │
    ▼
停用 (Deactivation)
    │
    ▼
卸载 (Unloading)
```

### 5.2 关键文件

| 文件 | 位置 | 说明 |
|------|------|------|
| `loader.ts` | `src/plugins/loader.ts` (2188 行) | 插件加载器 |
| `registry.ts` | `src/plugins/registry.ts` | 插件注册表 |
| `manifest.ts` | `src/plugins/manifest.ts` | 插件清单解析 |
| `install.ts` | `src/plugins/install.ts` | 插件安装 |
| `update.ts` | `src/plugins/update.ts` | 插件更新 |
| `uninstall.ts` | `src/plugins/uninstall.ts` | 插件卸载 |

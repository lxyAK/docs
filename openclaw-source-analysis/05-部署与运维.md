## 一、部署方式

OpenClaw 支持三种主要部署方式：

| 方式 | 场景 | 优点 | 缺点 |
|------|------|------|------|
| **CLI 本地运行** | 个人开发、测试 | 简单、快速 | 后台运行需手动管理 |
| **Daemon 守护进程** | 服务器长期运行 | 自动重启、系统集成 | 需要配置服务 |
| **Docker 容器** | 云部署、隔离环境 | 隔离、可移植 | 需要 Docker |

## 二、Docker 部署

### 2.1 Docker Compose

```yaml
services:
  openclaw-gateway:
    image: openclaw:local
    environment:
      HOME: /home/node
      TERM: xterm-256color
      OPENCLAW_GATEWAY_TOKEN: ${OPENCLAW_GATEWAY_TOKEN:-}
      OPENCLAW_ALLOW_INSECURE_PRIVATE_WS: ${OPENCLAW_ALLOW_INSECURE_PRIVATE_WS:-}
      CLAUDE_AI_SESSION_KEY: ${CLAUDE_AI_SESSION_KEY:-}
      CLAUDE_WEB_SESSION_KEY: ${CLAUDE_WEB_SESSION_KEY:-}
      CLAUDE_WEB_COOKIE: ${CLAUDE_WEB_COOKIE:-}
      TZ: ${OPENCLAW_TZ:-UTC}
    volumes:
      - ${OPENCLAW_CONFIG_DIR}:/home/node/.openclaw
      - ${OPENCLAW_WORKSPACE_DIR}:/home/node/.openclaw/workspace
      # - /var/run/docker.sock:/var/run/docker.sock  # 启用沙箱
    ports:
      - "${OPENCLAW_GATEWAY_PORT:-18789}:18789"
      - "${OPENCLAW_BRIDGE_PORT:-18790}:18790"
    init: true
    restart: unless-stopped
    command:
      [
        "node",
        "dist/index.js",
        "gateway",
        "--bind",
        "${OPENCLAW_GATEWAY_BIND:-lan}",
        "--port",
        "18789",
      ]
    healthcheck:
      test:
        [
          "CMD",
          "node",
          "-e",
          "fetch('http://127.0.0.1:18789/healthz').then((r)=>process.exit(r.ok?0:1)).catch(()=>process.exit(1))",
        ]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 20s

  openclaw-cli:
    image: openclaw:local
    network_mode: "service:openclaw-gateway"
    cap_drop:
      - NET_RAW
      - NET_ADMIN
    security_opt:
      - no-new-privileges:true
    environment:
      HOME: /home/node
      TERM: xterm-256color
      OPENCLAW_GATEWAY_TOKEN: ${OPENCLAW_GATEWAY_TOKEN:-}
      OPENCLAW_ALLOW_INSECURE_PRIVATE_WS: ${OPENCLAW_ALLOW_INSECURE_PRIVATE_WS:-}
      BROWSER: echo
      CLAUDE_AI_SESSION_KEY: ${CLAUDE_AI_SESSION_KEY:-}
      CLAUDE_WEB_SESSION_KEY: ${CLAUDE_WEB_SESSION_KEY:-}
      CLAUDE_WEB_COOKIE: ${CLAUDE_WEB_COOKIE:-}
      TZ: ${OPENCLAW_TZ:-UTC}
    volumes:
      - ${OPENCLAW_CONFIG_DIR}:/home/node/.openclaw
      - ${OPENCLAW_WORKSPACE_DIR}:/home/node/.openclaw/workspace
    stdin_open: true
    tty: true
    init: true
    entrypoint: ["node", "dist/index.js"]
    depends_on:
      - openclaw-gateway
```

### 2.2 端口说明

| 端口 | 用途 |
|------|------|
| 18789 | Gateway WebSocket/HTTP API |
| 18790 | Bridge 端口（可选） |

### 2.3 启动命令

```bash
# 构建镜像
docker build -t openclaw:local .

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f openclaw-gateway

# 进入 CLI
docker-compose run --rm openclaw-cli status
```

## 三、Daemon 守护进程

### 3.1 支持的服务管理器

| 平台 | 服务管理器 | 文件 |
|------|-----------|------|
| Linux | systemd | `/etc/systemd/system/openclaw.service` |
| macOS | launchd | `~/Library/LaunchAgents/ai.openclaw.gateway.plist` |
| Windows | Task Scheduler | 任务计划程序 |

### 3.2 Daemon 模块

位置：`src/daemon/` (~30+ 文件)

| 文件 | 说明 |
|------|------|
| `service.ts` | 服务管理主入口 |
| `systemd.ts` | systemd 集成 |
| `launchd.ts` | launchd 集成 |
| `schtasks.ts` | Windows 任务计划集成 |
| `constants.ts` | 常量定义 |
| `paths.ts` | 路径管理 |
| `diagnostics.ts` | 诊断 |

### 3.3 Daemon 命令

```bash
# 安装为服务
openclaw daemon install

# 启动服务
openclaw daemon start

# 停止服务
openclaw daemon stop

# 重启服务
openclaw daemon restart

# 查看状态
openclaw daemon status

# 查看日志
openclaw daemon logs

# 卸载服务
openclaw daemon uninstall
```

## 四、CLI 运维命令

### 4.1 状态与诊断

```bash
# 查看状态
openclaw status

# 详细状态
openclaw status --all

# 健康检查
openclaw doctor

# 深度诊断
openclaw doctor --deep

# 配置诊断
openclaw doctor config

# 安全审计
openclaw doctor security
```

### 4.2 配置管理

```bash
# 查看配置
openclaw config list

# 设置配置
openclaw config set agents.defaults.model gpt-4

# 移除配置
openclaw config unset agents.defaults.model

# 编辑配置（打开编辑器）
openclaw config edit

# 备份配置
openclaw backup create

# 恢复配置
openclaw backup restore <backup-id>
```

### 4.3 会话管理

```bash
# 列出会话
openclaw sessions list

# 查看会话详情
openclaw sessions show <session-key>

# 删除会话
openclaw sessions delete <session-key>

# 导出会话
openclaw sessions export <session-key>

# 清理旧会话
openclaw sessions cleanup
```

### 4.4 插件管理

```bash
# 列出插件
openclaw plugins list

# 安装插件
openclaw plugins install <plugin-id>

# 更新插件
openclaw plugins update <plugin-id>

# 卸载插件
openclaw plugins uninstall <plugin-id>
```

### 4.5 密钥管理

```bash
# 列出密钥（不显示值）
openclaw secrets list

# 设置密钥
openclaw secrets set OPENAI_API_KEY sk-...

# 删除密钥
openclaw secrets delete OPENAI_API_KEY

# 密钥审计
openclaw secrets audit
```

## 五、健康检查与监控

### 5.1 健康检查端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/healthz` | GET | 健康检查 |
| `/readyz` | GET | 就绪检查 |
| `/metrics` | GET | 指标（Prometheus 格式） |

### 5.2 健康检查实现

```javascript
// Docker healthcheck
test: [
  "CMD",
  "node",
  "-e",
  "fetch('http://127.0.0.1:18789/healthz')" +
  ".then((r)=>process.exit(r.ok?0:1))" +
  ".catch(()=>process.exit(1))",
]
```

### 5.3 日志位置

| 部署方式 | 日志位置 |
|---------|---------|
| CLI 本地 | `~/.openclaw/logs/` |
| Daemon | systemd journal / Console.app |
| Docker | `docker logs openclaw-gateway` |

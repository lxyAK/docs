## 一、整体链路概览

```
┌─────────────┐
│   用户      │
│  (飞书/TG)  │
└──────┬──────┘
       │ 1. 用户发送消息
       ▼
┌─────────────────────────────────────────┐
│   Channel Plugin                        │
│   (extensions/feishu/src/channel.ts)  │
│   - 接收 Webhook / WebSocket           │
│   - 解析平台特定格式                   │
│   - 转换为标准化消息                   │
└──────┬──────────────────────────────────┘
       │ 2. 标准化消息
       ▼
┌─────────────────────────────────────────┐
│   Gateway (gateway/server-methods/)   │
│   - 路由到 chat.ts                     │
│   - Session 查找/创建                  │
│   - 权限检查（白名单）                 │
└──────┬──────────────────────────────────┘
       │ 3. 调用 auto-reply
       ▼
┌─────────────────────────────────────────┐
│   Auto-Reply Engine                    │
│   (src/auto-reply/)                    │
│   - 命令检测 (/status, /focus)        │
│   - 指令解析 ([[reply_to]])            │
│   - 调用 get-reply.ts                  │
└──────┬──────────────────────────────────┘
       │ 4. 调用 Agent
       ▼
┌─────────────────────────────────────────┐
│   Agent (src/agents/)                  │
│   - 构建上下文                          │
│   - 选择模型 + Provider                 │
│   - 发送给 LLM                          │
│   - 处理流式响应                        │
│   - 工具调用 (如果需要)                 │
│   - 失败重试 / Fallback                 │
└──────┬──────────────────────────────────┘
       │ 5. 回复
       ▼
┌─────────────────────────────────────────┐
│   Auto-Reply → Outbound                │
│   - 回复格式化                          │
│   - 消息去重                            │
│   - Threading 处理                      │
└──────┬──────────────────────────────────┘
       │ 6. 发送到 Channel
       ▼
┌─────────────────────────────────────────┐
│   Channel Plugin → Outbound            │
│   - 转换为平台特定格式                 │
│   - 调用平台 API 发送                   │
└──────┬──────────────────────────────────┘
       │ 7. 用户看到回复
       ▼
┌─────────────┐
│   用户      │
└─────────────┘
```

## 二、入站消息处理

### 2.1 入口：Channel Plugin

每个 Channel Plugin 有自己的入站处理方式：

**Webhook 方式**（飞书、Discord 等）：
```
平台 → Webhook → OpenClaw → Channel Plugin
```

**WebSocket 方式**（Telegram、Signal 等）：
```
平台 ← WebSocket → OpenClaw → Channel Plugin
```

### 2.2 关键文件：`gateway/server-methods/chat.ts` (2094 行)

这是入站消息的核心处理文件：

| 步骤 | 说明 |
|------|------|
| 1 | 解析 `inbound` 消息（附件、元数据） |
| 2 | 权限检查（白名单、DM 策略） |
| 3 | Session 查找/创建 |
| 4 | 调用 `auto-reply/get-reply.ts` |
| 5 | 处理流式输出 |
| 6 | 发送回 Channel |

### 2.3 消息标准化

Channel Plugin 将平台特定格式转换为**标准化消息**：

```typescript
type StandardizedMessage = {
  text: string;
  attachments?: Attachment[];
  sender: {
    id: string;
    name?: string;
  };
  channel: {
    id: string;
    type: "dm" | "group";
  };
  timestamp: Date;
  thread?: {
    id: string;
  };
};
```

### 3.2 单次尝试：`pi-embedded-runner/run/attempt.ts` (2384 行)

每次重试都是一个独立的 `attempt`：

| 步骤 | 文件 | 说明 |
|------|------|------|
| 1 | `prepareContext()` | 准备上下文 |

| 4 | `sendToProvider()` | 发送给 Provider |

| 6 | `handleToolCalls()` | 处理工具调用 |

## 四、出站消息发送

### 4.1 回复分发：`auto-reply/reply/reply-dispatcher.ts`

决定回复怎么发：

- **立即发送** vs **延迟发送**
- **单条消息** vs **多条消息**
- **普通消息** vs **话题回复**
- **当前 Channel** vs **跨 Channel**

### 4.2 消息去重：`auto-reply/reply/reply-payloads-dedupe.ts`

避免重复发送相同内容：

```typescript
type DedupeKey = {
  sessionKey: string;
  contentHash: string;
  timestamp: number;
};
```

### 4.3 Threading 处理：`auto-reply/reply/reply-threading.ts`

处理话题/线程回复：

- 自动创建话题
- 查找话题上下文
- 话题内消息顺序

## 五、关键模块详解

### 5.1 Auto-Reply 引擎

位置：`src/auto-reply/` (~200+ 文件)

```
src/auto-reply/
├── get-reply.ts              # 主入口
├── dispatch.ts               # 分发
├── commands/                 # 内置命令
│   ├── commands.ts           # 命令注册
│   ├── commands-status.ts    # /status
│   ├── commands-config.ts    # /config
│   ├── commands-session.ts   # /session
│   └── ...
├── reply/                    # 回复处理
│   ├── reply.ts              # 回复主逻辑
│   ├── reply-dispatcher.ts   # 回复分发
│   ├── reply-payloads.ts     # 回复 Payload
│   ├── reply-threading.ts    # 话题处理
│   └── ...
├── model.ts                  # 模型选择
├── heartbeat.ts              # 心跳处理
└── ...
```

### 5.2 命令检测

内置命令检测：

| 命令 | 触发 | 说明 |
|------|------|------|
| `/status` | `message.startsWith("/status")` | 状态查询 |
| `/config` | `message.startsWith("/config")` | 配置管理 |
| `/session` | `message.startsWith("/session")` | 会话管理 |
| `/focus` | `message.startsWith("/focus")` | 专注模式 |
| `/doctor` | `message.startsWith("/doctor")` | 诊断工具 |

### 5.3 指令解析

支持的指令格式：

```
[[reply_to:12345]]      - 回复特定消息
[[reply_to_current]]    - 回复当前消息
[[focus:goal]]          - 设置专注目标
[[thinking:high]]       - 设置思考模式
```

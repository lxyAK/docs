## 一、安全体系全景

OpenClaw 安全策略分为 **7 个层级**，从外到内逐层防护：

```
┌───────────────────────────────────────────────────────┐
│  第 1 层：DM / 群组访问控制                            │
│  谁 能 和 Bot 对话？                                   │
│  (dm-policy-shared.ts)                                │
├───────────────────────────────────────────────────────┤
│  第 2 层：上下文可见性                                 │
│  对 话 中 能 看 到 什 么 ？                           │
│  (context-visibility.ts)                              │
├───────────────────────────────────────────────────────┤
│  第 3 层：工具策略管线                                 │
│  Agent 能 用 哪 些 工 具 ？                           │
│  (tool-policy-pipeline.ts)                            │
├───────────────────────────────────────────────────────┤
│  第 4 层：Owner-Only 工具守卫                          │
│  敏 感 工 具 只 有 主 人 能 用 ？                      │
│  (tool-policy.ts)                                     │
├───────────────────────────────────────────────────────┤
│  第 5 层：沙箱隔离                                    │
│  工 具 执 行 在 哪 里 运 行 ？                         │
│  (sandbox/)                                           │
├───────────────────────────────────────────────────────┤
│  第 6 层：子 Agent 限制                               │
│  子 Agent 能 用 什 么 ？                               │
│  (pi-tools.policy.ts)                                 │
├───────────────────────────────────────────────────────┤
│  第 7 层：Skill 安全扫描                              │
│  安 装 的 Skill 安 全 吗 ？                           │
│  (skill-scanner.ts)                                   │
└───────────────────────────────────────────────────────┘
```

## 二、安全审计系统

### 2.1 审计类型定义

```typescript
// src/security/audit.types.ts

type SecurityAuditSeverity = "info" | "warn" | "critical";

type SecurityAuditFinding = {
  checkId: string;           // 检查 ID
  severity: SecurityAuditSeverity;  // 严重级别
  title: string;             // 标题
  detail: string;            // 详情
  remediation?: string;      // 修复建议
};

type SecurityAuditReport = {
  ts: number;                // 时间戳
  summary: {                 // 汇总
    critical: number;
    warn: number;
    info: number;
  };
  findings: SecurityAuditFinding[];  // 所有发现
  deep?: {                   // 深度审计结果
    gateway?: {
      attempted: boolean;
      url: string | null;
      ok: boolean;
      error: string | null;
    };
  };
};
```

### 2.2 审计范围

| 检查项 | 说明 | 严重级别 |
|--------|------|---------|
| **危险配置标志** | `dangerouslyDisableDeviceAuth`、`allowInsecureAuth` 等 | critical |
| **DM 策略** | `dmPolicy=open` 允许任何人私聊 | critical |
| **群组策略** | `groupPolicy=open` 允许任何群使用 | critical |
| **文件权限** | 配置文件/密钥文件的文件系统权限 | warn |
| **未配置凭证** | Channel 缺少必要凭证 | warn |
| **插件代码安全** | 安装的插件/Skill 中的危险代码 | critical |
| **Gateway 探测** | 深度模式下探测 Gateway 安全性 | warn |

### 2.3 危险配置标志检测

```typescript
// src/security/dangerous-config-flags.ts

// 自动检测的危险配置：
- gateway.controlUi.allowInsecureAuth = true
- gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback = true
- gateway.controlUi.dangerouslyDisableDeviceAuth = true
- hooks.gmail.allowUnsafeExternalContent = true
- tools.exec.applyPatch.workspaceOnly = false
- 插件自定义危险标志 (通过 configContracts.dangerousFlags)
```

### 2.4 审计命令

```bash
# 快速审计
openclaw doctor security

# 深度审计（包含 Gateway 探测 + 插件代码扫描）
openclaw doctor security --deep

# 文件系统权限审计
openclaw doctor security --fs
```

## 三、工具策略管线

### 3.1 七步策略管线

工具策略通过 **管线（Pipeline）** 逐层过滤，每一步可以进一步缩小工具范围：

```
全部工具
    │
    ▼ Step 1: Profile 策略
    │ tools.profile (minimal/coding/messaging/full)
    │
    ▼ Step 2: Provider Profile 策略
    │ tools.byProvider.profile
    │
    ▼ Step 3: 全局策略
    │ tools.allow / tools.deny
    │
    ▼ Step 4: 全局 Provider 策略
    │ tools.byProvider.allow / tools.byProvider.deny
    │
    ▼ Step 5: Agent 策略
    │ agents.<id>.tools.allow / tools.deny
    │
    ▼ Step 6: Agent Provider 策略
    │ agents.<id>.tools.byProvider.allow
    │
    ▼ Step 7: 群组策略
      group tools.allow / tools.deny
```

### 3.2 工具 Profile

OpenClaw 预定义了 4 种工具 Profile：

| Profile | 包含工具 | 适用场景 |
|---------|---------|---------|
| **minimal** | 最少工具 | 纯对话场景 |
| **coding** | read/write/edit/exec/web_search/web_fetch/... | 代码开发 |
| **messaging** | message/browser/sessions_send/... | 消息处理 |
| **full** | 全部工具（无限制） | 完整功能 |

### 3.3 工具分组

```
group:openclaw    → web_search, web_fetch, code_execution, ...
group:fs          → read, write, edit, apply_patch
group:runtime     → exec, process, code_execution
group:web         → web_search, web_fetch
group:memory      → memory_store, memory_recall, memory_forget
group:sessions    → sessions_spawn, sessions_send, sessions_list, ...
group:ui          → canvas, browser
group:messaging   → message, tts
group:automation  → cron
group:nodes       → nodes
group:agents      → subagents, agents_list
group:media       → image, pdf, tts
```

### 3.4 配置示例

```json
{
  "tools": {
    "profile": "coding",
    "allow": ["read", "write", "exec"],
    "deny": ["apply_patch"]
  }
}
```

## 四、DM / 群组访问控制

### 4.1 DM 策略（私聊）

| 策略 | 说明 | 安全级别 |
|------|------|---------|
| `disabled` | 禁止所有私聊 | 最高 |
| `allowlist` | 只允许白名单用户 | 高 |
| `pairing` | 需要配对授权（默认） | 中 |
| `open` | 允许所有人私聊 | 最低 |

### 4.2 群组策略

| 策略 | 说明 | 安全级别 |
|------|------|---------|
| `disabled` | 禁止所有群组 | 最高 |
| `allowlist` | 只允许白名单群组 | 高 |
| `open` | 允许所有群组 | 最低 |

### 4.3 访问决策流程

```
收到消息
    │
    ├── 是群组消息？
    │   ├── groupPolicy = disabled → 🚫 拒绝
    │   ├── groupPolicy = allowlist + 群在白名单 → ✅ 允许
    │   ├── groupPolicy = allowlist + 群不在白名单 → 🚫 拒绝
    │   └── groupPolicy = open → ✅ 允许
    │
    └── 是私聊消息？
        ├── dmPolicy = disabled → 🚫 拒绝
        ├── dmPolicy = open → ✅ 允许
        ├── dmPolicy = allowlist + 用户在白名单 → ✅ 允许
        ├── dmPolicy = pairing + 已配对 → ✅ 允许
        ├── dmPolicy = pairing + 未配对 → 🔗 需要配对
        └── dmPolicy = allowlist + 用户不在白名单 → 🚫 拒绝
```

## 五、上下文可见性

### 5.1 可见性模式

```typescript
// src/security/context-visibility.ts

type ContextVisibilityMode = "all" | "allowlist" | "allowlist_quote";
type ContextVisibilityKind = "history" | "thread" | "quote" | "forwarded";
```

| 模式 | 说明 |
|------|------|
| `all` | 所有人可见所有上下文 |
| `allowlist` | 只有白名单用户可见 |
| `allowlist_quote` | 白名单用户可见 + 引用消息始终可见 |

### 5.2 决策逻辑

```typescript
function evaluateSupplementalContextVisibility(params) {
  if (params.mode === "all") {
    return { include: true, reason: "mode_all" };
  }
  if (params.senderAllowed) {
    return { include: true, reason: "sender_allowed" };
  }
  if (params.mode === "allowlist_quote" && params.kind === "quote") {
    return { include: true, reason: "quote_override" };
  }
  return { include: false, reason: "blocked" };
}
```

## 六、Skill 安全扫描

### 6.1 扫描规则

`src/security/skill-scanner.ts` 对安装的 Skill 进行静态代码扫描：

#### 行级规则（LINE_RULES）

| 规则 ID | 严重级别 | 说明 | 检测模式 |
|---------|---------|------|---------|
| `dangerous-exec` | **critical** | Shell 命令执行 | `exec(/exec\|execSync\|spawn\|spawnSync/)` + 需要 `child_process` 上下文 |
| `dynamic-code-execution` | **critical** | 动态代码执行 | `eval(` 或 `new Function(` |
| `crypto-mining` | **critical** | 加密货币挖矿 | `stratum+tcp`, `coinhive`, `xmrig` |
| `suspicious-network` | warn | 可疑 WebSocket 连接 | 非标准端口的 `new WebSocket()` |

#### 源码级规则（SOURCE_RULES）

| 规则 ID | 严重级别 | 说明 | 检测模式 |
|---------|---------|------|---------|
| `potential-exfiltration` | warn | 文件读取 + 网络发送 = 数据泄露 | `readFileSync` + `fetch/post` |
| `obfuscated-code` | warn | 十六进制混淆代码 | 连续 6+ 个 `\x` 编码 |
| `obfuscated-code` | warn | 大 Base64 + 解码 = 混淆 | `atob/Buffer.from` + 200+ 字符 Base64 |
| `env-harvesting` | **critical** | 环境变量 + 网络发送 = 凭证窃取 | `process.env` + `fetch/post` |

### 6.2 扫描限制

- 最多扫描 500 个文件
- 单文件最大 1MB
- 只扫描 `.js/.ts/.mjs/.cjs/.mts/.cts/.jsx/.tsx` 文件

## 八、密钥安全

### 8.1 时序安全比较

防止时序攻击（Timing Attack）：

```typescript
// src/security/secret-equal.ts

function safeEqualSecret(provided: string, expected: string): boolean {
  // 使用 SHA-256 哈希 + timingSafeEqual 防止时序攻击
  const hash = (s: string) => createHash("sha256").update(s).digest();
  return timingSafeEqual(hash(provided), hash(expected));
}
```

### 8.2 密钥存储

- 密钥加密存储在 `~/.openclaw/secrets/`
- 支持系统密钥链（macOS Keychain / Linux Secret Service）
- 配置文件中密钥自动脱敏

### 9.2 叶子节点额外禁止

```typescript
const SUBAGENT_TOOL_DENY_LEAF = [
  "subagents",           // 子 Agent 编排
  "sessions_list",       // 会话列表
  "sessions_history",    // 会话历史
```

### 9.3 层级关系

```
主 Agent (depth=0)
    │
    ├── 编排子 Agent (depth=1, maxDepth≥2)
    │   │
    │   │  可用：sessions_spawn, subagents, sessions_list, sessions_history
    │   │  禁用：gateway, agents_list, whatsapp_login, session_status, cron, sessions_send
    │   │
    │   └── 叶子子 Agent (depth ≥ maxSpawnDepth)
    │       │
    │       │  可用：read, write, edit, exec, web_search, web_fetch, ...
    │       │  额外禁用：subagents, sessions_list, sessions_history, sessions_spawn
    │
    └── ...
```

## 十、Gateway HTTP 工具黑名单

### 10.1 默认禁止的 HTTP 工具

通过 Gateway HTTP API (`POST /tools/invoke`) 默认禁止的工具：

```typescript
// src/security/dangerous-tools.ts

const DEFAULT_GATEWAY_HTTP_TOOL_DENY = [
  "exec",            // 命令执行 (RCE)
  "spawn",           // 子进程 (RCE)
  "shell",           // Shell 执行 (RCE)
  "fs_write",        // 文件写入
  "fs_delete",       // 文件删除
  "fs_move",         // 文件移动
  "apply_patch",     // 补丁应用
  "sessions_spawn",  // Agent 生成 (RCE)
  "sessions_send",   // 消息注入
  "cron",            // 定时任务操控
  "gateway",         // Gateway 重配置
  "nodes",           // 节点命令中继
  "whatsapp_login",  // 交互式登录
];
```

### 10.2 为什么禁止

| 工具 | 风险 |
|------|------|
| `exec/spawn/shell` | **远程代码执行 (RCE)** — 攻击者可执行任意命令 |
| `fs_write/fs_delete/fs_move` | **文件系统破坏** — 可修改/删除任意文件 |
| `sessions_spawn` | **间接 RCE** — 通过生成 Agent 执行代码 |
| `sessions_send` | **消息注入** — 跨会话注入消息 |
| `cron` | **持久化攻击** — 创建定时恶意任务 |
| `gateway` | **Gateway 重配置** — 修改安全设置 |
| `nodes` | **横向移动** — 在配对设备上执行命令 |

## 总结

OpenClaw 的安全策略是一个**纵深防御体系**：

1. **入口控制** — DM/群组策略决定谁能和 Bot 对话
2. **上下文控制** — 可见性策略决定能看到什么
3. **工具控制** — 策略管线决定能用哪些工具
4. **Owner 守卫** — 敏感工具只有主人能用
5. **沙箱隔离** — 工具执行在隔离环境中
6. **子 Agent 限制** — 层级递减的工具权限
7. **静态扫描** — Skill 安装前的代码安全检查
8. **密钥保护** — 时序安全比较 + 加密存储

每一层都有独立的防护机制，即使某一层被突破，下一层仍然提供保护。

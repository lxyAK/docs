## 一、配置系统

### 1.1 配置文件位置

```
~/.openclaw/
├── config.json               # 主配置文件
├── config.json.backup        # 备份
├── sessions/                 # 会话存储
│   ├── transcripts/          # Transcripts
│   └── store.json            # Session Store
├── secrets/                  # 密钥存储（加密）
├── plugins/                  # 插件数据
└── hooks/                    # 钩子数据
```

### 1.2 配置加载流程

```
默认值 (Defaults)
    ↑
环境变量 (ENV Vars)
    ↑
配置文件 (config.json)
    ↑
命令行参数 (CLI Flags)
```

### 1.3 核心配置文件

位置：`src/config/`

| 文件 | 说明 |
|------|------|
| `config.ts` | 配置加载主入口 |
| `schema.base.generated.ts` (27117 行) | 自动生成的基础 Schema |
| `zod-schema.providers-core.ts` (1614 行) | Provider 配置 Schema |
| `io.ts` (1813 行) | 配置 IO（读写文件） |
| `config-schema.ts` | 配置 Schema 定义 |

### 1.4 配置层级

配置按层级合并，优先级从低到高：

```typescript
// 1. 默认值（代码中硬编码）
const defaults = {
  agents: {
    defaults: {
      model: "claude-3-5-sonnet-20241022",
      provider: "anthropic"
    }
  }
};

// 2. 环境变量
process.env.OPENCLAW_AGENTS__DEFAULTS__MODEL = "gpt-4";

// 3. 配置文件 ~/.openclaw/config.json
{
  "agents": {
    "defaults": {
      "model": "gpt-4-turbo"
    }
  }
}

// 4. 命令行参数
openclaw --agents.defaults.model=gpt-4o
```

## 二、密钥管理

### 2.1 密钥存储

密钥加密存储在 `~/.openclaw/secrets/`：

```
~/.openclaw/secrets/
├── store.json              # 加密的密钥存储
└── .key                    # 加密密钥（可选，否则用系统密钥链）
```

### 2.2 密钥类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Provider API Key | AI 模型提供商密钥 | `OPENAI_API_KEY` |
| Channel 凭证 | 消息平台凭证 | `TELEGRAM_BOT_TOKEN` |
| OAuth Token | OAuth 授权令牌 | `GITHUB_TOKEN` |
| 自定义密钥 | 用户自定义密钥 | `MY_CUSTOM_SECRET` |

### 2.3 密钥管理模块

位置：`src/secrets/` (~107 个文件)

| 文件 | 说明 |
|------|------|
| `storage-scan.ts` | 密钥存储扫描 |
| `resolve.ts` | 密钥解析 |
| `audit.ts` | 密钥审计 |
| `runtime.ts` | 运行时密钥管理 |
| `channel-env-vars.ts` | Channel 环境变量 |
| `provider-env-vars.ts` | Provider 环境变量 |

### 2.4 密钥解析流程

```
配置引用密钥
    │
    ▼
查找密钥存储
    │
    ▼
解密密钥
    │
    ▼
注入环境变量 / 直接使用
```

## 三、安全审计

### 3.1 安全模块

位置：`src/security/`

| 文件 | 说明 |
|------|------|
| `audit.ts` | 安全审计主入口 |
| `audit.deep.runtime.ts` | 深度审计 |
| `audit-fs.ts` | 文件系统审计 |
| `audit-tool-policy.ts` | 工具策略审计 |
| `dangerous-tools.ts` | 危险工具列表 |
| `dangerous-config-flags.ts` | 危险配置标志 |
| `skill-scanner.ts` | Skill 扫描器 |
| `channel-metadata.ts` | 通道元数据 |
| `secret-equal.ts` | 安全比较（防止时序攻击） |

### 3.2 审计级别

| 级别 | 说明 |
|------|------|
| `quick` | 快速审计（默认） |
| `deep` | 深度审计（扫描所有文件） |
| `fs` | 文件系统审计 |

### 3.3 审计命令

```bash
openclaw doctor security          # 快速安全审计
openclaw doctor security --deep   # 深度安全审计
openclaw doctor security --fs     # 文件系统审计
```

## 四、工具策略

### 4.1 工具策略模块

位置：`src/agents/tool-policy.ts` (~1500 行)

### 4.2 策略层级

```
全局策略 (Global Policy)
    ↑
Agent 策略 (Agent Policy)
    ↑
Session 策略 (Session Policy)
    ↑
单次调用策略 (Per-call Policy)
```

### 4.3 工具白名单/黑名单

```json
{
  "agents": {
    "defaults": {
      "toolPolicy": {
        "allowlist": ["read", "write", "edit"],
        "denylist": ["exec"]
      }
    }
  }
}
```

### 4.4 执行审批

敏感工具需要用户审批：

```typescript
type ApprovalRequest = {
  tool: string;
  args: Record<string, unknown>;
  reason: string;
  risk: "low" | "medium" | "high";
};
```

> 版本：2026.4.11 | 分析日期：2026-04-11
> 仓库：https://github.com/openclaw/openclaw
> 技术栈：TypeScript + Node.js 22+ (ESM) | 构建：tsdown | 测试：Vitest

---

## 一、项目定位

OpenClaw 是一个 **多通道 AI 网关**（Multi-channel AI Gateway），核心能力：

- 连接 30+ 消息平台（飞书、Telegram、Discord、Signal、微信等）
- 对接 20+ AI 模型提供商（OpenAI、Anthropic、Google、DeepSeek 等）
- 支持工具调用（Tool Use）、MCP 协议、子 Agent 编排
- 可作为 CLI 工具、常驻守护进程（Daemon）或 Docker 容器运行

```
用户消息 → [Channel] → [Gateway] → [Agent/LLM] → [Tool Execution] → 回复 → [Channel] → 用户
```

## 五、启动链路

```
1. openclaw.mjs (Node.js 入口 wrapper)
   ├── 检查 Node.js 版本 ≥ 22.14.0
   ├── 启用编译缓存 (module.enableCompileCache)
   └── 导入 dist/entry.js

2. src/entry.ts
   ├── 解析 CLI 参数
   ├── 处理 Windows argv 兼容
   ├── 判断子命令 (gateway / agent / talk / ...)
   └── 分发到对应 CLI 模块

3. CLI 分发
   ├── gateway → 启动 WebSocket/HTTP 网关服务
   ├── agent  → 启动 Agent 交互循环（连接 Gateway）
   ├── talk   → 启动 Talk Mode（终端对话）
   └── 其他子命令 (secrets/status/doctor/...)
```

## 八、关键设计决策

1. **ESM Only** — 纯 ES Module，不使用 CommonJS
2. **TypeScript Strict** — 全量 strict 模式
3. **Zod Schema** — 配置系统使用 Zod 做运行时校验
4. **Plugin-First** — 所有 Channel/Provider 都以插件形式存在
5. **Session-Based** — 每个对话有独立的 Session Key 和 Transcripts
6. **Embedded Agent** — Agent 嵌入在 Gateway 进程中运行（非独立进程）
7. **Tool Policy Pipeline** — 工具执行有完整的安全策略管线
8. **Contract Testing** — 插件间通过 Contract Test 保证接口兼容性

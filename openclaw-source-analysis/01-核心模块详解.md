## 一、Gateway 模块（网关）

### 1.1 模块定位

Gateway 是 OpenClaw 的**核心协调层**，主要职责：

- 提供 WebSocket/HTTP API 接口
- 接收来自 Channel 的用户消息
- 调用 Agent 处理并执行工具
- 将回复路由回 Channel
- 管理 Session 生命周期

### 1.2 目录结构

```
src/gateway/
├── server-methods/          # 核心 API 方法（60+ 个）
│   ├── agent.ts              # Agent 执行 / spawn
│   ├── chat.ts               # 聊天处理（2094 行）
│   ├── sessions.ts           # Session 管理（1655 行）
│   ├── send.ts               # 消息发送
│   ├── channels.ts           # 通道管理
│   ├── config.ts             # 配置读写
│   ├── secrets.ts            # 密钥管理
│   ├── doctor.ts             # 诊断工具
│   ├── models.ts             # 模型管理
│   ├── plugins.ts            # 插件管理
│   ├── skills.ts             # 技能管理
│   ├── nodes.ts              # 节点管理
│   ├── exec.ts               # 工具执行审批
│   └── ... 其他 50+ 个方法
├── method-scopes.ts          # API 权限范围
├── http-endpoint-helpers.ts  # HTTP 端点辅助
└── ... 其他
```

### 1.3 Server Methods 概览

Gateway 通过 `server-methods` 暴露完整功能，按分类：

| 分类 | 方法 | 说明 |
|------|------|------|
| **Agent** | `agent`, `agents`, `agent-job` | Agent 执行、列表、后台任务 |
| **Chat** | `chat` | 核心聊天处理（2094 行） |
| **Session** | `sessions`, `sessions.runtime` | Session 查询、运行时管理 |
| **Send** | `send` | 发送消息到 Channel |
| **Channel** | `channels` | 通道配置与状态 |
| **Config** | `config` | 配置读写 |
| **Secrets** | `secrets` | 密钥管理（加密存储） |
| **Models** | `models` | 模型列表与选择 |
| **Plugins** | `plugins` | 插件安装/更新/删除 |
| **Skills** | `skills` | 技能加载与使用 |
| **Nodes** | `nodes`, `nodes-pending` | 配对设备管理 |
| **Exec** | `exec-approvals` | 工具执行审批 |
| **Health** | `health` | 健康检查 |
| **Logs** | `logs` | 日志查询 |
| **Cron** | `cron` | 定时任务管理 |
| **Usage** | `usage` | 使用统计 |
| **Doctor** | `doctor` | 诊断工具 |
| **System** | `system`, `restart-request` | 系统级操作 |
| **Talk** | `talk` | 语音对话模式 |
| **Web** | `web` | Web 界面相关 |
| **Wizard** | `wizard` | 设置向导 |

### 1.4 Gateway 消息处理流程

```
用户消息 → Channel Plugin → Gateway
                              │
                              ▼
    ┌─────────────────────────────────────────────┐
    │ gateway/server-methods/chat.ts             │
    ├─────────────────────────────────────────────┤
    │ 1. 解析 inbound 消息                       │
    │ 2. 验证权限 / 白名单                        │
    │ 3. 查找/创建 Session                        │
    │ 4. 调用 Agent (auto-reply)                 │
    │ 5. 处理回复流 (streaming)                   │
    │ 6. 发送回 Channel                           │
    └─────────────────────────────────────────────┘
                              │
                              ▼
                         Agent 执行
```

## 二、Agents 模块（AI 代理）

### 2.1 模块定位

Agents 是 OpenClaw 的**智能核心**，负责：

- 与 LLM Provider 通信（OpenAI/Anthropic/Google 等）
- 管理工具调用（Tool Use）
- 处理流式输出（Streaming）
- 管理上下文窗口（Context Window）
- 处理失败重试与 Fallback
- 子 Agent 编排与隔离

### 2.2 目录结构

```
src/agents/
├── pi-embedded/              # PI Agent 嵌入运行器
│   ├── runner.ts             # 主运行器 (2221 行)
│   ├── run/
│   │   └── attempt.ts        # 单次运行尝试 (2384 行)
│   ├── helpers/
│   │   └── errors.ts         # 错误处理 (1511 行)
│   └── ...
├── bash-tools.ts             # Bash 工具 (1778 行)
├── openclaw-tools.ts         # OpenClaw 内置工具
├── subagent-*.ts             # 子 Agent 管理
│   ├── subagent-spawn.ts      # 生成子 Agent
│   ├── subagent-registry.ts   # 子 Agent 注册表
│   ├── subagent-announce.ts   # 子 Agent 通知
│   └── ...
├── model-*.ts                # 模型相关
│   ├── models-config.ts        # 模型配置
│   ├── model-selection.ts      # 模型选择
│   ├── model-fallback.ts       # 模型降级
│   └── ...
├── provider-*.ts              # Provider 通信
│   ├── anthropic-transport-stream.ts
│   ├── openai-transport-stream.ts
│   ├── google-transport-stream.ts
│   └── ...
├── tool-*.ts                 # 工具相关
│   ├── tool-catalog.ts         # 工具目录
│   ├── tool-policy.ts          # 工具策略
│   ├── tool-display.ts         # 工具展示
│   └── ...
├── context.ts                # 上下文管理
├── system-prompt.ts          # 系统提示词
├── memory-search.ts          # 记忆搜索
├── skills.ts                 # 技能集成
├── identity.ts               # Agent 身份
├── workspace.ts              # 工作区
└── ... 其他 400+ 个文件
```

### 2.3 Agent 运行链路

```
用户消息 → Gateway
              │
              ▼
    ┌───────────────────────────────────┐
    │  auto-reply/get-reply.ts         │
    └───────────────────────────────────┘
              │
              ▼
    ┌───────────────────────────────────┐
    │  agents/pi-embedded-runner.ts    │
    │  (2221 行核心运行器)              │
    ├───────────────────────────────────┤
    │ 1. 准备上下文                      │
    │ 2. 选择模型 + Provider            │
    │ 3. 构建请求 Payload                │
    │ 4. 发送给 LLM                      │
    │ 5. 处理流式响应                    │
    │ 6. 执行工具调用 (如果需要)         │
    │ 7. 重试 / Fallback (失败时)        │
    │ 8. 返回最终回复                    │
    └───────────────────────────────────┘
```

### 2.4 核心类与函数

| 文件/函数 | 说明 |
|-----------|------|
| `pi-embedded-runner.ts` (2221 行) | PI Agent 主运行器，完整的 LLM 对话循环 |
| `pi-embedded-runner/run/attempt.ts` (2384 行) | 单次运行尝试（重试时多次调用） |
| `bash-tools.exec.ts` (1778 行) | `exec` 工具实现（命令执行） |
| `openclaw-tools.ts` | OpenClaw 内置工具（read/write/edit/exec 等） |
| `models-config.ts` | 模型配置（Provider 发现、模型列表） |
| `model-fallback.ts` | 模型降级策略（主模型失败时自动切换） |
| `tool-policy.ts` | 工具策略（安全检查、白名单） |
| `subagent-spawn.ts` | 生成子 Agent（隔离执行） |
| `system-prompt.ts` | 系统提示词构建 |

## 三、Channels 模块（消息通道）

### 3.1 模块定位

Channels 是 OpenClaw 的**通信桥接层**，负责：

- 抽象不同消息平台的差异
- 处理入站消息（接收）
- 处理出站消息（发送）
- 会话绑定（Channel ↔ Session）
- 认证与权限（白名单）

### 3.2 目录结构

```
src/channels/
├── plugins/                  # Channel Plugin 框架
│   ├── index.ts              # 插件注册表
│   ├── catalog.ts            # 插件目录
│   ├── channel-config.ts     # 通道配置
│   ├── types.ts              # 类型定义
│   ├── setup-wizard-helpers.ts (1627 行)  # 设置向导
│   └── ...
├── session.ts                # 会话管理
├── registry.ts               # 通道注册表
├── targets.ts                # 目标路由
├── ids.ts                    # ID 生成
├── chat-meta.ts              # 聊天元数据
├── channel-config.ts         # 通道配置
├── allowlist-match.ts        # 白名单匹配
├── mention-gating.ts         # @提及门控
├── typing.ts                 # 输入状态
├── web/                      # Web 通道
└── transport/                # 传输层
```

### 3.3 Channel Plugin 生命周期

```
加载 (Loader)
    │
    ▼
发现 (Discovery)
    │
    ▼
设置 (Setup Wizard)
    │
    ▼
激活 (Activation)
    │
    ▼
运行 (Runtime)
    │
    ├── 接收入站消息
    │       │
    │       ▼
    │   Gateway → Agent
    │
    └── 发送出站消息
            │
            ▼
        用户看到回复
```

位置：`extensions/feishu/`

```
extensions/feishu/
├── openclaw.plugin.json      # 插件清单
├── channel-entry.ts          # 通道入口
├── src/
│   ├── channel.ts            # 飞书通道实现
│   ├── thread-bindings.ts    # 话题绑定
│   ├── outbound.ts           # 消息发送
│   └── ...
├── setup-entry.ts            # 设置入口
├── api.ts                    # API 封装
└── skills/                   # 飞书技能
    └── feishu-*/SKILL.md
```

## 四、Sessions 模块（会话管理）

### 4.1 模块定位

Sessions 是 OpenClaw 的**状态管理层**，负责：

- Session Key 生成与解析
- Transcript 存储与加载
- 会话生命周期管理
- 会话标签与元数据

### 4.2 目录结构

```
src/sessions/
├── session-id.ts             # Session ID 生成
├── session-key.ts            # Session Key 解析
├── session-label.ts          # 会话标签
├── session-lifecycle-events.ts  # 生命周期事件
├── transcript-events.ts      # Transcript 事件
├── input-provenance.ts       # 输入来源
├── send-policy.ts            # 发送策略
├── model-overrides.ts        # 模型覆盖
└── ...
```

### 4.3 Session Key 格式

Session Key 是 OpenClaw 的**会话唯一标识**，支持多种形态：

| 形态 | 示例 | 用途 |
|------|------|------|
| 主会话 | `user:ou_12345` | 用户主会话 |
| 标签会话 | `user:ou_12345#my-project` | 带标签的会话 |
| 显式会话 | `explicit:abc123` | 显式创建的会话 |
| 子 Agent 会话 | `subagent:def456` | 子 Agent 隔离会话 |

## 五、Config 模块（配置系统）

### 5.1 模块定位

Config 是 OpenClaw 的**配置管理层**，负责：

- 配置加载、验证、合并
- 配置 Schema 定义（Zod）
- 配置 IO（读写文件）
- 配置迁移与兼容

### 5.2 目录结构

```
src/config/
├── config.ts                 # 主配置加载
├── schema.base.generated.ts  # 基础 Schema（27117 行，自动生成）
├── zod-schema.providers-core.ts (1614 行)  # Provider 配置 Schema
├── io.ts                     # 配置 IO（1813 行）
├── sessions/                 # Session 配置
│   ├── store.ts              # Session 存储
│   └── paths.ts              # 路径解析
├── merge-patch.proto-pollution.test.ts
└── ...
```

### 5.3 配置层级

```
默认值 (Defaults)
    ↑
环境变量 (ENV)
    ↑
配置文件 (config.json)
    ↑
命令行参数 (CLI Flags)
```

## 七、关键数据流转总结

```
┌─────────────────────────────────────────────────────────────┐
│  用户（飞书/TG/...）                                       │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 消息
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  Channels Module（通道插件）                                │
│  - 接收入站消息                                             │
│  - 解析消息格式                                             │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 标准化消息
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  Gateway Module（网关）                                     │
│  - 路由请求                                                 │
│  - Session 管理                                             │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 调用 Agent
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  Auto-Reply Engine（自动回复引擎）                          │
│  - 命令解析                                                 │
│  - 指令处理                                                 │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 执行 LLM 调用
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  Agents Module（Agent）                                     │
│  - 与 LLM Provider 通信                                     │
│  - 工具调用                                                 │
│  - 流处理                                                   │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 回复
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  Gateway → Channel Plugin                                   │
│  - 发送回复                                                 │
└────────────────┬────────────────────────────────────────────┘
                 │
                 │ 显示给用户
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  用户（看到回复）                                           │
└─────────────────────────────────────────────────────────────┘
```

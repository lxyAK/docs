> 版本：main (2026-04-14) | 分析日期：2026-04-14
> 仓库：https://github.com/NousResearch/hermes-agent
> 技术栈：Python 3.11+ | 框架：OpenAI SDK + Fire CLI

---

## 一、项目定位

Hermes Agent 是 Nous Research 开源的 **自我改进 AI Agent**，核心差异点：

- **闭环学习**：从经验中自动创建 Skills，使用中改进，跨会话搜索记忆
- **多模型支持**：OpenRouter（200+ 模型）、Nous Portal、OpenAI、Anthropic、z.ai/GLM、Kimi、MiniMax 等
- **多通道网关**：Telegram、Discord、Slack、WhatsApp、Signal、飞书、微信、Matrix 等
- **多终端后端**：本地、Docker、SSH、Modal（Serverless）、Daytona、Singularity
- **OpenClaw 兼容**：内置 `hermes claw migrate` 迁移工具

```
用户消息 → [CLI/Gateway] → [run_agent.py 主循环] → [LLM API] → [工具执行] → 回复
                                    ↓                    ↑
                              [记忆/上下文]          [Skill/MCP]
```

## 五、与 OpenClaw 的架构对比

| 维度 | OpenClaw | Hermes Agent |
|------|----------|-------------|
| **语言** | TypeScript / Node.js | Python |
| **定位** | 多通道 AI 网关 | 自我改进 AI Agent |
| **核心循环** | Gateway → Agent → Channel | CLI/Gateway → AIAgent → LLM API |
| **工具系统** | 内置 + MCP | 内置 Registry + MCP + Skills |
| **记忆** | MEMORY.md + 外部 SDK | MemoryManager + 插件化后端 + FTS5 搜索 |
| **平台数** | 30+ | 10+ |
| **模型** | 20+ Provider | OpenRouter 200+ / 直连 |
| **学习闭环** | 无 | Skill 自动创建 + 自改进 + 跨会话搜索 |
| **RL 训练** | 无 | 内置 Atropos 环境 |
| **配置** | YAML + ENV | config.yaml + ENV |
| **子Agent** | sessions_spawn | delegate_tool（并行子Agent） |
| **MCP** | ✅ | ✅ |
| **飞书** | 一等公民 | 平台适配器 |
| **代码量** | ~613K 行 TS | ~389K 行 Python |

## 六、关键设计哲学

1. **单文件巨石**：`run_agent.py` 10K+ 行，承载完整的 Agent 主循环——Python 风格的"一个文件搞定"
2. **注册模式**：工具通过 `registry.register()` 在 import 时自注册，零配置发现
3. **插件化扩展**：记忆、上下文引擎均可通过插件替换，放文件即生效
4. **Skill 即程序记忆**：区别于声明式记忆（MEMORY.md），Skill 是过程性知识
5. **Serverless 友好**：Modal/Daytona 后端支持空闲休眠、按需唤醒
6. **安全纵深**：Prompt 注入扫描 → 命令审批 → 沙箱隔离 → 路径安全检查

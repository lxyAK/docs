## 二、本机部署（快速上手）

### 2.1 系统要求

| 平台 | 最低版本 | 推荐版本 |
|------|---------|---------|
| **Linux** | Ubuntu 20.04+ | Ubuntu 22.04+ |
| **macOS** | 12 (Monterey) | 14 (Sonoma) |
| **WSL2** | Windows 10/11 | Windows 11 |
| **Android** | Termux (F-Droid) | Termux 0.118+ |

**软件依赖**：
- Python 3.11+
- Git
- curl

### 2.2 一键安装

```bash
# 一键安装脚本（推荐）
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 安装后重新加载 shell
source ~/.bashrc    # 或 source ~/.zshrc

# 验证安装
hermes --version
```

**高级选项**：
```bash
# 自定义分支安装
curl -fsSL ... | bash -s -- --branch main

# 跳过虚拟环境
curl -fsSL ... | bash -s -- --no-venv

# 跳过初始化向导
curl -fsSL ... | bash -s -- --skip-setup
```

### 2.3 手动安装（开发者模式）

```bash
# 1. 克隆仓库
git clone https://github.com/NousResearch/hermes-agent.git
cd hermes-agent

# 2. 安装 uv 包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. 创建虚拟环境
uv venv venv --python 3.11
source venv/bin/activate

# 4. 安装依赖
uv pip install -e ".[all,dev]"

# 5. 运行测试
python -m pytest tests/ -q
```

### 2.4 初始化配置

```bash
# 交互式配置向导（推荐）
hermes setup

# 或分步配置
hermes model           # 选择 LLM Provider 和模型
hermes tools           # 配置工具启用/禁用
hermes config set      # 设置单个配置项
hermes doctor          # 诊断问题
```

### 2.5 目录结构

```
~/.hermes/
├── config.yaml         # 配置文件
├── .env                # 环境变量（API Keys）
├── credentials/        # 加密凭证存储
├── skills/             # 技能目录
│   ├── user-created/   # 用户创建的技能
│   └── hub-installed/  # Skills Hub 安装
├── sessions/           # 会话历史
├── cron/               # 定时任务
│   ├── jobs.json       # 任务定义
│   └── outputs/        # 执行输出
└── hermes-agent/       # 安装目录（可选）
```

## 三、Docker 部署

### 3.1 Docker Compose 方案（推荐）

创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  hermes:
    image: nousresearch/hermes-agent:latest
    container_name: hermes-agent
    restart: unless-stopped
    
    # 端口映射
    ports:
      - "8765:8765"  # ACP Server
      - "8080:8080"  # Web UI
    
    # 持久化存储
    volumes:
      - hermes-config:/root/.hermes
      - ./workspace:/workspace:rw
    
    # 环境变量
    environment:
      - HERMES_HOME=/root/.hermes
      - OPENROUTER_API_KEY=${OPENROUTER_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - DISCORD_BOT_TOKEN=${DISCORD_BOT_TOKEN}
      - FEISHU_APP_ID=${FEISHU_APP_ID}
      - FEISHU_APP_SECRET=${FEISHU_APP_SECRET}
    
    # 安全选项
    cap_drop:
      - ALL
    security_opt:
      - no-new-privileges:true

# 持久化卷
volumes:
  hermes-config:
    driver: local
```

### 3.2 启动与管理

```bash
# 1. 创建 .env 文件
cat > .env << 'EOF'
OPENROUTER_API_KEY=sk-or-v1-...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
TELEGRAM_BOT_TOKEN=...
DISCORD_BOT_TOKEN=...
FEISHU_APP_ID=cli_...
FEISHU_APP_SECRET=...
EOF

# 2. 启动服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f hermes

# 4. 进入容器配置
docker-compose exec hermes hermes setup

# 5. 停止服务
docker-compose down

# 6. 升级镜像
docker-compose pull
docker-compose up -d
```

### 3.3 自定义 Docker 镜像

创建 `Dockerfile`：

```dockerfile
# 基础镜像
FROM python:3.11-slim-bookworm

# 元数据
LABEL maintainer="Nous Research"
LABEL description="Hermes Agent - Self-improving AI Agent"

# 环境变量
ENV HERMES_HOME=/root/.hermes
ENV UV_SYSTEM_PYTHON=1
ENV PATH="$PATH:/root/.local/bin"

# 安装系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    curl \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# 安装 uv
RUN curl -LsSf https://astral.sh/uv/install.sh | sh

# 设置工作目录
WORKDIR /app

# 复制代码
COPY . .

# 安装 Python 依赖
RUN uv pip install -e ".[all]"

# 暴露端口
EXPOSE 8765 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=5s --start-period=60s \
    CMD uv run hermes doctor || exit 1

# 入口点
ENTRYPOINT ["uv", "run", "hermes"]
CMD ["--help"]
```

### 3.4 Docker 运行模式

```bash
# CLI 模式（交互式）
docker run -it --rm \
    -v hermes-config:/root/.hermes \
    -v $(pwd):/workspace \
    nousresearch/hermes-agent:latest

# Gateway 模式（后台服务）
docker run -d --name hermes-gateway \
    -v hermes-config:/root/.hermes \
    -p 8765:8765 \
    -e TELEGRAM_BOT_TOKEN=... \
    nousresearch/hermes-agent:latest \
    gateway start

# ACP Server 模式
docker run -d --name hermes-acp \
    -v hermes-config:/root/.hermes \
    -p 8765:8765 \
    nousresearch/hermes-agent:latest \
    -m acp_adapter
```

## 四、配置优化

### 4.1 config.yaml 核心配置

```yaml
# ~/.hermes/config.yaml
model:
  provider: openrouter
  name: anthropic/claude-sonnet-4
  
  # Smart Model Routing
  routing:
    enabled: true
    cheap: anthropic/claude-haiku-4
    strong: anthropic/claude-sonnet-4

terminal:
  env: docker  # local|docker|modal|ssh|daytona
  docker:
    image: hermes-agent:latest
    network: none

memory:
  provider: holographic  # builtin|mem0|holographic|...

delegation:
  max_concurrent_children: 3

cron:
  enabled: true

context:
  engine: compressor
  threshold_percent: 0.75

gateway:
  telegram:
    enabled: true
    token: "${TELEGRAM_BOT_TOKEN}"
  feishu:
    enabled: true
    app_id: "${FEISHU_APP_ID}"
    app_secret: "${FEISHU_APP_SECRET}"
```

### 4.2 环境变量配置

```bash
# ~/.hermes/.env
OPENROUTER_API_KEY=sk-or-v1-...
```

## 六、常见问题

| 问题 | 解决方案 |
|------|---------|
| Python 版本不对 | 使用 `uv venv venv --python 3.11` 指定 |
| 依赖安装失败 | 升级 uv: `uv self update` |
| API Key 不生效 | 检查 `~/.hermes/.env` 权限 |
| Gateway 无法连接 | 检查防火墙和端口映射 |
| Docker 权限问题 | 添加用户到 docker 组：`usermod -aG docker $USER` |

---

## 七、安全建议

1. **API Key 保护**：使用加密存储，不要提交到 Git
2. **命令审批**：生产环境使用 `approval: ask` 模式
3. **沙箱隔离**：优先使用 Docker 终端后端
4. **网络限制**：容器默认无网络访问
5. **定期更新**：`hermes update` 保持最新版本

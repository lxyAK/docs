## 二、本机部署（快速上手）

### 2.1 系统要求

| 平台 | 最低版本 | 推荐版本 |
|------|---------|---------|
| **Linux** | Ubuntu 20.04+ | Ubuntu 22.04+ |
| **macOS** | 12 (Monterey) | 14 (Sonoma) |
| **WSL2** | Windows 10/11 | Windows 11 |
| **Android** | Termux (F-Droid) | Termux 0.118+ |

**软件依赖**：
- Python 3.11+
- Git
- curl

### 2.2 一键安装

```bash
# 一键安装脚本（推荐）
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 安装后重新加载 shell
source ~/.bashrc    # 或 source ~/.zshrc

# 验证安装
hermes --version
```

**高级选项**：
```bash
# 自定义分支安装
curl -fsSL ... | bash -s -- --branch main

# 跳过虚拟环境
curl -fsSL ... | bash -s -- --no-venv

# 跳过初始化向导
curl -fsSL ... | bash -s -- --skip-setup
```

### 2.3 手动安装（开发者模式）

```bash
# 1. 克隆仓库
git clone https://github.com/NousResearch/hermes-agent.git
cd hermes-agent

# 2. 安装 uv 包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. 创建虚拟环境
uv venv venv --python 3.11
source venv/bin/activate

# 4. 安装依赖
uv pip install -e ".[all,dev]"

# 5. 运行测试
python -m pytest tests/ -q
```

### 2.4 初始化配置

```bash
# 交互式配置向导（推荐）
hermes setup

# 或分步配置
hermes model           # 选择 LLM Provider 和模型
hermes tools           # 配置工具启用/禁用
hermes config set      # 设置单个配置项
hermes doctor          # 诊断问题
```

### 2.5 目录结构

```
~/.hermes/
├── config.yaml         # 配置文件
├── .env                # 环境变量（API Keys）
├── credentials/        # 加密凭证存储
├── skills/             # 技能目录
│   ├── user-created/   # 用户创建的技能
│   └── hub-installed/  # Skills Hub 安装
├── sessions/           # 会话历史
├── cron/               # 定时任务
│   ├── jobs.json       # 任务定义
│   └── outputs/        # 执行输出
└── hermes-agent/       # 安装目录（可选）
```

## 三、Docker 部署

### 3.1 官方镜像说明

Hermes Agent 官方提供了 Dockerfile，核心特性：
- **基于 debian:13.4**，内置 Python 3.13
- **使用 uv** 作为包管理器
- **gosu** 权限降级（默认 UID 10000）
- **Playwright** 浏览器预装
- **WhatsApp Bridge** 支持
- **自动初始化配置**（config.yaml、.env、SOUL.md）

### 3.2 官方 Docker Compose 方案（推荐）

创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  hermes:
    build:
      context: https://github.com/NousResearch/hermes-agent.git#main
    container_name: hermes-agent
    restart: unless-stopped
    
    # UID/GID 映射（可选，匹配主机用户避免权限问题）
    environment:
      - HERMES_UID=1000
      - HERMES_GID=1000
      - HERMES_HOME=/opt/data
      
      # API Keys
      - OPENROUTER_API_KEY=${OPENROUTER_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      
      # Gateway 配置
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - DISCORD_BOT_TOKEN=${DISCORD_BOT_TOKEN}
      - FEISHU_APP_ID=${FEISHU_APP_ID}
      - FEISHU_APP_SECRET=${FEISHU_APP_SECRET}
    
    # 持久化存储
    volumes:
      - hermes-data:/opt/data
      - ./workspace:/opt/data/workspace:rw

# 持久化卷
volumes:
  hermes-data:
    driver: local
```

### 3.3 启动与管理

```bash
# 1. 创建 .env 文件
cat > .env << 'EOF'
OPENROUTER_API_KEY=sk-or-v1-...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
TELEGRAM_BOT_TOKEN=...
DISCORD_BOT_TOKEN=...
FEISHU_APP_ID=cli_...
FEISHU_APP_SECRET=...
EOF

# 2. 启动服务（构建镜像 + 启动）
docker-compose up -d --build

# 3. 查看日志
docker-compose logs -f hermes

# 4. 进入容器执行命令
docker-compose exec hermes hermes setup

# 5. 进入容器交互式 shell
docker-compose exec hermes bash

# 6. 停止服务
docker-compose down

# 7. 升级并重新构建
docker-compose down
docker-compose up -d --build
```

### 3.4 官方 Docker 运行模式

```bash
# CLI 模式（交互式）
docker run -it --rm \
    -v hermes-data:/opt/data \
    -v $(pwd):/opt/data/workspace \
    nousresearch/hermes-agent:latest

# Gateway 模式（后台服务，通过 Telegram/Discord 等消息平台交互）
docker run -d --name hermes-gateway \
    -v hermes-data:/opt/data \
    -e TELEGRAM_BOT_TOKEN=... \
    nousresearch/hermes-agent:latest \
    chat --platform telegram

# 覆盖 UID/GID 以匹配主机权限
docker run -d --name hermes \
    -v hermes-data:/opt/data \
    -e HERMES_UID=$(id -u) \
    -e HERMES_GID=$(id -g) \
    nousresearch/hermes-agent:latest
```

### 3.5 目录结构（容器内）

```
/opt/hermes/          # 安装目录（只读）
├── .venv/            # Python 虚拟环境
├── skills/           # 内置技能
├── scripts/
└── docker/
    └── entrypoint.sh

/opt/data/            # 数据目录（持久化卷）
├── .env              # 环境变量
├── config.yaml       # 配置
├── SOUL.md           # 人格文件
├── cron/             # 定时任务
├── sessions/         # 会话历史
├── logs/             # 日志
├── hooks/            # Gateway 钩子脚本
├── memories/         # 持久化记忆存储
├── plans/            # 计划文件
├── skins/            # 皮肤/主题
├── skills/           # 用户技能
├── workspace/        # 工作空间（可挂载到主机）
└── home/             # 子进程 HOME（git/ssh/npm 等）
```

## 四、配置优化

### 4.1 config.yaml 核心配置

```yaml
# ~/.hermes/config.yaml（对应源码 cli-config.yaml.example）

# =============================================================================
# 模型配置
# =============================================================================
model:
  # 默认模型（也可通过 --model 参数覆盖）
  default: "anthropic/claude-opus-4.6"

  # 推理提供商选择：
  #   "auto"         - 根据凭证自动检测（默认）
  #   "openrouter"   - OpenRouter（需要 OPENROUTER_API_KEY）
  #   "anthropic"    - 直连 Anthropic API（需要 ANTHROPIC_API_KEY）
  #   "nous"         - Nous Portal OAuth
  #   "gemini"       - Google AI Studio（需要 GOOGLE_API_KEY）
  #   "zai"          - 智谱 GLM（需要 GLM_API_KEY）
  #   "custom"       - 任意 OpenAI 兼容端点（设 base_url）
  provider: "auto"
  base_url: "https://openrouter.ai/api/v1"

  # Smart Model Routing（可选）
  # 用廉价模型处理简单请求，主力模型处理复杂请求
  # smart_model_routing:
  #   enabled: true
  #   max_simple_chars: 160
  #   max_simple_words: 28
  #   cheap_model:
  #     provider: openrouter
  #     model: google/gemini-2.5-flash

# =============================================================================
# 终端工具配置
# =============================================================================
terminal:
  backend: "local"     # local|docker|singularity|modal|ssh|daytona
  cwd: "."
  timeout: 180
  lifetime_seconds: 300
  docker_mount_cwd_to_workspace: false
  # Docker 后端示例：
  # backend: "docker"
  # docker_image: "nikolaik/python-nodejs:python3.11-nodejs20"
  # docker_mount_cwd_to_workspace: true

# =============================================================================
# 上下文压缩（自动压缩长对话）
# =============================================================================
compression:
  enabled: true              # 启用自动压缩（默认 true）
  threshold: 0.50            # 达到上下文窗口 50% 时触发
  target_ratio: 0.20         # 保留 20% 作为最近对话尾部
  protect_last_n: 20         # 始终保留最近 20 条消息

# =============================================================================
# 持久化记忆
# =============================================================================
memory:
  memory_enabled: true       # Agent 个人笔记
  user_profile_enabled: true # 用户画像
  memory_char_limit: 2200    # ~800 tokens
  user_char_limit: 1375      # ~500 tokens
  nudge_interval: 10         # 每 10 轮提醒保存记忆
  flush_min_turns: 6         # 退出时至少 6 轮才触发记忆保存

# =============================================================================
# 子 Agent 委派
# =============================================================================
delegation:
  max_iterations: 50                                    # 每个 child 最大工具调用轮次
  default_toolsets: ["terminal", "file", "web"]         # 子 Agent 默认工具集

# =============================================================================
# Agent 行为
# =============================================================================
agent:
  max_turns: 60              # 每次对话最大工具调用迭代数
  verbose: false
  reasoning_effort: "medium" # xhigh|high|medium|low|minimal|none

# =============================================================================
# 会话重置（消息平台）
# =============================================================================
session_reset:
  mode: both         # both|idle|daily|none
  idle_minutes: 1440 # 24 小时无活动重置
  at_hour: 4         # 每天凌晨 4 点重置

# Gateway 平台配置通过 .env 环境变量管理：
#   TELEGRAM_BOT_TOKEN, FEISHU_APP_ID, FEISHU_APP_SECRET 等
# 不在 config.yaml 中配置
```

## 六、常见问题

| 问题 | 解决方案 |
|------|---------|
| Python 版本不对 | 使用 `uv venv venv --python 3.11` 指定 |
| 依赖安装失败 | 升级 uv: `uv self update` |
| API Key 不生效 | 检查 `~/.hermes/.env` 权限 |
| Gateway 无法连接 | 检查防火墙和端口映射 |
| Docker 权限问题 | 添加用户到 docker 组：`usermod -aG docker $USER` |

---

## 七、安全建议

1. **API Key 保护**：使用加密存储，不要提交到 Git
2. **命令审批**：生产环境使用 `approval: ask` 模式
3. **沙箱隔离**：优先使用 Docker 终端后端
4. **网络限制**：容器默认无网络访问
5. **定期更新**：Gateway 模式使用 `/update` 命令；CLI 模式重新运行安装脚本

## 八、升级与卸载

```bash
# 升级（注意：仅 Gateway 消息平台模式可用 /update 命令）
# CLI 模式请重新运行安装脚本或 git pull + uv pip install -e .

# Docker 升级（重新构建镜像）
docker-compose up -d --build

# 卸载
# 注意：会删除所有配置和数据
rm -rf ~/.hermes
```

## 二、本机部署（快速上手）

### 2.1 系统要求

| 平台 | 最低版本 | 推荐版本 |
|------|---------|---------|
| **Linux** | Ubuntu 20.04+ | Ubuntu 22.04+ |
| **macOS** | 12 (Monterey) | 14 (Sonoma) |
| **WSL2** | Windows 10/11 | Windows 11 |
| **Android** | Termux (F-Droid) | Termux 0.118+ |

**软件依赖**：
- Python 3.11+
- Git
- curl

### 2.2 一键安装

```bash
# 一键安装脚本（推荐）
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 安装后重新加载 shell
source ~/.bashrc    # 或 source ~/.zshrc

# 验证安装
hermes --version
```

**高级选项**：
```bash
# 自定义分支安装
curl -fsSL ... | bash -s -- --branch main

# 跳过虚拟环境
curl -fsSL ... | bash -s -- --no-venv

# 跳过初始化向导
curl -fsSL ... | bash -s -- --skip-setup
```

### 2.3 手动安装（开发者模式）

```bash
# 1. 克隆仓库
git clone https://github.com/NousResearch/hermes-agent.git
cd hermes-agent

# 2. 安装 uv 包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. 创建虚拟环境
uv venv venv --python 3.11
source venv/bin/activate

# 4. 安装依赖
uv pip install -e ".[all,dev]"

# 5. 运行测试
python -m pytest tests/ -q
```

### 2.4 初始化配置

```bash
# 交互式配置向导（推荐）
hermes setup

# 或分步配置
hermes model           # 选择 LLM Provider 和模型
hermes tools           # 配置工具启用/禁用
hermes config set      # 设置单个配置项
hermes doctor          # 诊断问题
```

### 2.5 目录结构

```
~/.hermes/
├── config.yaml         # 配置文件
├── .env                # 环境变量（API Keys）
├── credentials/        # 加密凭证存储
├── skills/             # 技能目录
│   ├── user-created/   # 用户创建的技能
│   └── hub-installed/  # Skills Hub 安装
├── sessions/           # 会话历史
├── cron/               # 定时任务
│   ├── jobs.json       # 任务定义
│   └── outputs/        # 执行输出
└── hermes-agent/       # 安装目录（可选）
```

## 三、Docker 部署

### 3.1 官方镜像说明

Hermes Agent 官方提供了 Dockerfile，核心特性：
- **基于 debian:13.4**
- **使用 uv** 作为包管理器
- **gosu** 权限降级（默认 UID 10000）
- **Playwright** 浏览器预装
- **WhatsApp Bridge** 支持
- **自动初始化配置**（config.yaml、.env、SOUL.md）

### 3.2 官方 Docker Compose 方案（推荐）

创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  hermes:
    build:
      context: https://github.com/NousResearch/hermes-agent.git#main
    container_name: hermes-agent
    restart: unless-stopped
    
    # UID/GID 映射（可选，匹配主机用户避免权限问题）
    environment:
      - HERMES_UID=1000
      - HERMES_GID=1000
      - HERMES_HOME=/opt/data
      
      # API Keys
      - OPENROUTER_API_KEY=${OPENROUTER_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      
      # Gateway 配置
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - DISCORD_BOT_TOKEN=${DISCORD_BOT_TOKEN}
      - FEISHU_APP_ID=${FEISHU_APP_ID}
      - FEISHU_APP_SECRET=${FEISHU_APP_SECRET}
    
    # 持久化存储
    volumes:
      - hermes-data:/opt/data
      - ./workspace:/opt/data/workspace:rw

# 持久化卷
volumes:
  hermes-data:
    driver: local
```

### 3.3 启动与管理

```bash
# 1. 创建 .env 文件
cat > .env << 'EOF'
OPENROUTER_API_KEY=sk-or-v1-...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
TELEGRAM_BOT_TOKEN=...
DISCORD_BOT_TOKEN=...
FEISHU_APP_ID=cli_...
FEISHU_APP_SECRET=...
EOF

# 2. 启动服务（构建镜像 + 启动）
docker-compose up -d --build

# 3. 查看日志
docker-compose logs -f hermes

# 4. 进入容器执行命令
docker-compose exec hermes hermes setup

# 5. 进入容器交互式 shell
docker-compose exec hermes bash

# 6. 停止服务
docker-compose down

# 7. 升级并重新构建
docker-compose down
docker-compose up -d --build
```

### 3.4 官方 Docker 运行模式

```bash
# CLI 模式（交互式）
docker run -it --rm \
    -v hermes-data:/opt/data \
    -v $(pwd):/opt/data/workspace \
    ghcr.io/nousresearch/hermes-agent:latest

# Gateway 模式（后台服务）
docker run -d --name hermes-gateway \
    -v hermes-data:/opt/data \
    -e TELEGRAM_BOT_TOKEN=... \
    ghcr.io/nousresearch/hermes-agent:latest \
    gateway start

# 覆盖 UID/GID 以匹配主机权限
docker run -d --name hermes \
    -v hermes-data:/opt/data \
    -e HERMES_UID=$(id -u) \
    -e HERMES_GID=$(id -g) \
    ghcr.io/nousresearch/hermes-agent:latest
```

### 3.5 目录结构（容器内）

```
/opt/hermes/          # 安装目录（只读）
├── .venv/            # Python 虚拟环境
├── skills/           # 内置技能
├── scripts/
└── docker/
    └── entrypoint.sh

/opt/data/            # 数据目录（持久化卷）
├── .env              # 环境变量
├── config.yaml       # 配置
├── SOUL.md           # 人格文件
├── cron/             # 定时任务
├── sessions/         # 会话历史
├── logs/             # 日志
├── skills/           # 用户技能
├── workspace/        # 工作空间（可挂载到主机）
└── home/             # 子进程 HOME（git/ssh/npm 等）
```

## 四、配置优化

### 4.1 config.yaml 核心配置

```yaml
# ~/.hermes/config.yaml
model:
  provider: openrouter
  name: anthropic/claude-sonnet-4
  
  # Smart Model Routing
  routing:
    enabled: true
    cheap: anthropic/claude-haiku-4
    strong: anthropic/claude-sonnet-4

terminal:
  env: docker  # local|docker|modal|ssh|daytona
  docker:
    image: hermes-agent:latest
    network: none

memory:
  provider: holographic  # builtin|mem0|holographic|...

delegation:
  max_concurrent_children: 3

cron:
  enabled: true

context:
  engine: compressor
  threshold_percent: 0.75

gateway:
  telegram:
    enabled: true
    token: "${TELEGRAM_BOT_TOKEN}"
  feishu:
    enabled: true
    app_id: "${FEISHU_APP_ID}"
    app_secret: "${FEISHU_APP_SECRET}"
```

### 4.2 环境变量配置

```bash
# ~/.hermes/.env
OPENROUTER_API_KEY=sk-or-v1-...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
TELEGRAM_BOT_TOKEN=...
DISCORD_BOT_TOKEN=...
FEISHU_APP_ID=cli_...
FEISHU_APP_SECRET=...

# 终端后端
TERMINAL_ENV=docker
TERMINAL_MAX_FOREGROUND_TIMEOUT=600

# 代理
HTTP_PROXY=http://localhost:7890
HTTPS_PROXY=http://localhost:7890
```

## 六、常见问题

| 问题 | 解决方案 |
|------|---------|
| Python 版本不对 | 使用 `uv venv venv --python 3.11` 指定 |
| 依赖安装失败 | 升级 uv: `uv self update` |
| API Key 不生效 | 检查 `~/.hermes/.env` 权限 |
| Gateway 无法连接 | 检查防火墙和端口映射 |
| Docker 权限问题 | 添加用户到 docker 组：`usermod -aG docker $USER` |

---

## 七、安全建议

1. **API Key 保护**：使用加密存储，不要提交到 Git
2. **命令审批**：生产环境使用 `approval: ask` 模式
3. **沙箱隔离**：优先使用 Docker 终端后端
4. **网络限制**：容器默认无网络访问
5. **定期更新**：`hermes update` 保持最新版本

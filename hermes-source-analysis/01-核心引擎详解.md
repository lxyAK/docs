## 一、AIAgent 主循环 (`run_agent.py`)

### 1.1 核心类 AIAgent

```
AIAgent
├── __init__()              # 初始化模型、工具、记忆、上下文
├── run_conversation()      # 主入口：用户消息 → 工具循环 → 最终回复
├── _call_llm()             # 调用 LLM API（含 failover、重试）
├── _handle_tool_call()     # 分发工具调用
├── _build_system_prompt()  # 组装系统提示词
├── _check_and_compress()   # 上下文压缩检查
├── _preflight_compression()# API 调用前预检
└── _display_response()     # 渲染响应
```

### 1.2 对话循环流程

```python
# 伪代码简化
def run_conversation(user_message):
    messages.append({"role": "user", "content": user_message})
    
    # 记忆预取
    memory_context = memory_manager.prefetch_all(user_message)
    
    while True:  # 工具循环
        # 1. 组装系统提示
        system_prompt = build_system_prompt() + memory_context + skills
        
        # 2. 上下文压缩检查
        if should_compress():
            messages = context_engine.compress(messages)
        
        # 3. 调用 LLM
        response = call_llm(messages, tools=tool_definitions)
        
        # 4. 检查是否有工具调用
        if response.tool_calls:
            for tool_call in response.tool_calls:
                result = handle_function_call(tool_call)
                messages.append(tool_result)
            continue  # 继续循环
        
        # 5. 无工具调用 → 完成
        break
    
    # 记忆同步
    memory_manager.sync_all(user_message, response)
    return response.content
```

### 1.3 关键机制

**Failover 与重试**：
- `classify_api_error()` 分类错误（速率限制/超时/上下文过长/服务器错误）
- `jittered_backoff()` 指数退避 + 随机抖动
- 自动切换备用模型/端点

**智能模型路由** (`smart_model_routing.py`)：
- 根据消息复杂度关键词（debug, implement, analyze...）选择 cheap vs strong 模型
- 可配置双模型策略，简单对话用廉价模型节省成本

**Prompt Caching** (`prompt_caching.py`)：
- Anthropic 原生缓存支持
- 自动标记静态部分（系统提示、技能索引）

## 三、工具调度层 (`model_tools.py`)

### 3.1 核心函数

| 函数 | 职责 |
|------|------|
| `get_tool_definitions()` | 从 Registry 收集工具 JSON Schema |
| `handle_function_call()` | 路由到具体工具 handler |
| `get_toolset_for_tool()` | 查询工具所属工具集 |
| `check_toolset_requirements()` | 检查工具集依赖是否满足 |

### 3.2 工具调用流程

```
LLM 返回 tool_calls
    ↓
handle_function_call(name, arguments)
    ↓
registry.get_tool(name) → ToolEntry
    ↓
检查: check_fn() → requires_env → 权限
    ↓
执行: handler(**arguments)
    ↓
结果持久化: maybe_persist_tool_result()
    ↓
Turn 预算检查: enforce_turn_budget()
    ↓
返回结果给 LLM
```

## 四、上下文压缩 (`agent/context_compressor.py`)

### 4.1 压缩策略

```
[受保护头部(3条)] + [待压缩中间] + [受保护尾部(6条)]
                         ↓
              辅助模型摘要（廉价/快速）
                         ↓
[受保护头部] + [摘要] + [受保护尾部]
```

### 4.2 关键设计

- **Token 预算尾保护**：基于 token 数而非固定消息数
- **工具输出裁剪**：LLM 摘要前先清除旧工具输出（廉价预过滤）
- **可缩放摘要预算**：`summary_tokens = max(2000, compressed_content * 0.20)`，上限 12K
- **摘要前缀隔离**：`[CONTEXT COMPACTION — REFERENCE ONLY]` 防止模型响应摘要中的旧问题
- **迭代摘要更新**：多次压缩保留信息

### 4.3 上下文引擎抽象 (`context_engine.py`)

```python
class ContextEngine(ABC):
    @property
    @abstractmethod
    def name(self) -> str: ...
    
    @abstractmethod
    def update_from_response(usage) -> None: ...
    
    @abstractmethod
    def should_compress(prompt_tokens) -> bool: ...
    
    @abstractmethod
    def compress(messages) -> List[Dict]: ...
    
    # 可选生命周期钩子
    def on_session_start(self): ...
    def on_session_end(self): ...
```

支持通过插件替换默认压缩引擎（如 LCM DAG 压缩）。

## 五、系统提示词组装 (`agent/prompt_builder.py`)

### 5.1 组装流水线

```
1. Agent 身份 (DEFAULT_AGENT_IDENTITY)
2. 平台提示 (PLATFORM_HINTS) — 根据 CLI/Telegram/Discord 等
3. SOUL.md 加载 + 安全扫描
4. 技能索引 (build_skills_system_prompt)
5. 上下文文件 (build_context_files_prompt) — AGENTS.md 等
6. 环境提示 (build_environment_hints)
7. 记忆上下文 (memory_context_block)
8. 工具使用引导 (TOOL_USE_ENFORCEMENT_GUIDANCE)
9. 模型特定引导 (Google/OpenAI/Anthropic 操作指导)
```

### 5.2 上下文文件安全扫描

在加载 AGENTS.md、SOUL.md 等文件前，扫描以下威胁模式：

| 模式 | 检测 |
|------|------|
| Prompt 注入 | `ignore previous instructions` |
| 隐瞒指令 | `do not tell the user` |
| 系统提示覆盖 | `system prompt override` |
| 不可见 Unicode | 零宽字符、方向控制符 |
| 凭证外泄 | `curl ... $API_KEY` |
| HTML 注入 | `<!-- secret -->`, `display:none` |

## 一、项目定位

Banana Slides 是一个 **AI 原生 PPT 生成应用**，基于 nano banana pro 🍌 模型驱动，核心能力：

- 一句话/大纲/页面描述自动生成完整 PPT
- 上传模板图片，AI 精准遵循风格
- 上传素材文件（PDF/Word/PPT），智能解析并融入生成
- 口头修改指定区域（Inpainting + OCR 可编辑导出）
- 支持多种 AI Provider 热切换（Gemini/OpenAI/Anthropic/国内厂商）

```
用户输入（主题/大纲/描述） → [AI 生成大纲] → [AI 生成页面描述] → [AI 生成幻灯片图片] → [可编辑导出 PPTX]
```

## 五、数据模型关系

```
Project (1) ──→ (N) Page
Project (1) ──→ (N) Task
Project (1) ──→ (N) Material
Project (1) ──→ (N) ReferenceFile
Page (1) ──→ (N) PageImageVersion
Settings (单例)
UserTemplate (独立)
```

### Project（项目）核心字段

| 字段 | 说明 |
|------|------|
| `creation_type` | 创建方式：`idea`（一句话）/ `outline`（大纲）/ `descriptions`（页面描述） |
| `template_image_path` | 模板图片路径 |
| `template_style` | 风格描述文本（无模板图时） |
| `image_aspect_ratio` | 图片比例：16:9、4:3、1:1 |
| `status` | 状态：DRAFT → GENERATING → COMPLETED |
| `export_*` | 导出设置（提取方法/Inpaint 方法/允许部分导出） |

### Page（页面）核心字段

| 字段 | 说明 |
|------|------|
| `outline_content` | 大纲内容（JSON） |
| `description_content` | 页面描述内容（JSON） |
| `generated_image_path` | 生成的原始图片路径 |
| `cached_image_path` | 压缩缩略图路径 |
| `status` | DRAFT → GENERATING → COMPLETED |

## 六、核心数据流

### 6.1 PPT 生成主流程

```
┌──────────────────────────────────────────────────────────────────┐
│  用户输入                                                        │
│  ├─ 一句话（idea）                                               │
│  ├─ 大纲文本（outline）                                          │
│  ├─ 页面描述（descriptions）                                      │
│  └─ 参考文件（PDF/Word/PPT）→ MinerU 解析 → Markdown            │
└──────────────┬───────────────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────────────┐
│  Step 1: 大纲生成                                                │
│  POST /api/projects/{id}/generate/outline/stream (SSE)          │
│  AI Provider (text) → 生成每页标题 + 要点                        │
└──────────────┬───────────────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────────────┐
│  Step 2: 页面描述生成                                            │
│  POST /api/projects/{id}/generate/descriptions/stream (SSE)     │
│  AI Provider (text) → 每页详细描述（标题/要点/视觉元素/排版等）   │
└──────────────┬───────────────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────────────┐
│  Step 3: 幻灯片图片生成                                          │
│  POST /api/projects/{id}/generate/images                        │
│  TaskManager (并发控制) → AI Provider (image) → 生成每页图片     │
│  支持：模板图风格遵循 / 自定义风格描述                            │
└──────────────┬───────────────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────────────┐
│  Step 4: 导出                                                    │
│  ├─ PPTX: 图片直接拼接（python-pptx）                            │
│  ├─ PDF: img2pdf 转换                                            │
│  ├─ 图片: 打包下载                                               │
│  └─ 可编辑 PPTX: OCR → 文字提取 → Inpainting → 可编辑文本框     │
└──────────────────────────────────────────────────────────────────┘
```

### 6.2 AI Provider 调用链

```
Controller → ai_service.py → ai_service_manager.py → Provider
                                                          ├─ GenAI Provider (Google Gemini)
                                                          ├─ OpenAI Provider
                                                          ├─ Anthropic Provider
                                                          └─ LazyLLM Provider (国内厂商)
                                                              ├─ Qwen (通义千问)
                                                              ├─ Doubao (豆包)
                                                              ├─ DeepSeek
                                                              ├─ GLM (智谱)
                                                              ├─ SiliconFlow
                                                              ├─ SenseNova (商汤)
                                                              └─ MiniMax
```

### 6.3 可编辑导出流程

```
图片 → OCR (百度) → 文字区域坐标映射 → Inpainting (去除文字) → 背景图
                                                           ├─ 百度 Inpainting
                                                           ├─ Gemini Inpainting
                                                           └─ 火山引擎 Inpainting
                                                    → python-pptx 组装可编辑文本框 → PPTX
```

## 七、API 设计

### RESTful 接口一览

| 模块 | 路径前缀 | 核心接口 |
|------|---------|---------|
| 项目 | `/api/projects` | CRUD + 大纲/描述/图片生成 + 改造 |
| 页面 | `/api/projects/{id}/pages` | CRUD + 单页生成/编辑/版本管理 |
| 模板 | `/api/templates` | 预设模板列表 |
| 用户模板 | `/api/user-templates` | 上传/列表/删除 |
| 素材 | `/api/materials` | 上传/AI 描述生成 |
| 导出 | `/api/projects/{id}/export` | PPTX/PDF/图片/可编辑 PPTX |
| 设置 | `/api/settings` | 全局设置 CRUD |
| 参考文件 | `/api/reference-files` | 上传/解析/内容获取 |
| 风格 | `/api/style` | 风格提取 |
| 访问控制 | `/api/access-code` | 口令验证 |

### 关键接口详情

**大纲生成（SSE 流式）**：
```
POST /api/projects/{id}/generate/outline/stream
→ SSE 事件流：outline_page / outline_complete / outline_error
```

**描述生成（SSE 流式）**：
```
POST /api/projects/{id}/generate/descriptions/stream
→ SSE 事件流：description_page / description_complete / description_error
```

**图片生成（异步任务）**：
```
POST /api/projects/{id}/generate/images
→ 返回 task_id，轮询 GET /api/projects/{id}/tasks/{task_id} 获取进度
```

**PPT 改造（Renovation）**：
```
POST /api/projects/renovation
→ 上传已有 PPT → 解析 → 重新生成各页图片
```

## 十、技术亮点

### 1. 多 Provider 热切换
text / image / image_caption 三种模型独立配置，每种可选不同的 AI Provider 和 API Key。

### 2. SSE 流式生成
大纲和描述生成使用 Server-Sent Events 流式推送，前端实时渲染进度。

### 3. 可编辑导出
OCR 文字提取 → 坐标映射 → Inpainting 去文字 → 背景图 + 可编辑文本框 = 真正可编辑的 PPTX。

### 4. 图片版本管理
每次重新生成图片保存为版本，支持回退到任意历史版本。

### 5. CLI 工具
`banana-cli` 支持批量任务、工作流编排、模板管理，便于自动化场景。

### 6. 访问控制
轻量级 HMAC 口令验证，适合小团队/个人部署场景。

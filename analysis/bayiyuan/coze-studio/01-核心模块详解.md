# Coze Studio 核心模块详解

## 领域模块概览

Coze Studio 的 Domain 层（领域层）包含 19 个核心领域模块，每个模块负责一个特定的业务领域。

```
domain/
├── agent/              # 智能体领域
├── app/                # 应用领域
├── connector/          # 连接器领域
├── conversation/       # 对话领域
├── datacopy/           # 数据复制领域
├── knowledge/          # 知识库领域
├── memory/             # 记忆领域
├── openauth/           # 开放认证领域
├── permission/         # 权限领域
├── plugin/             # 插件领域
├── prompt/             # 提示词领域
├── search/             # 搜索领域
├── shortcutcmd/        # 快捷命令领域
├── template/           # 模板领域
├── upload/             # 上传领域
├── user/               # 用户领域
└── workflow/           # 工作流领域
```

## 1. 工作流领域 (workflow)

工作流是 Coze Studio 的核心编排引擎，负责可视化流程的定义、执行和管理。

### 模块结构
```
domain/workflow/
├── interface.go                # 领域对外接口
├── component_interface.go      # 组件接口定义
├── entity/                     # 实体定义
│   ├── workflow.go            # 工作流实体
│   ├── node.go                # 节点实体
│   └── edge.go                # 边实体
├── service/                    # 领域服务
│   ├── workflow_service.go    # 工作流服务
│   └── executor_service.go    # 执行器服务
├── repository/                 # 仓储接口
│   └── workflow_repo.go       # 工作流仓储
├── variable/                   # 变量系统
│   ├── variable.go            # 变量定义
│   └── resolver.go            # 变量解析器
├── plugin/                     # 插件系统
│   └── plugin.go              # 插件接口
├── config/                     # 配置
└── internal/                   # 内部实现
```

### 核心实体

#### Workflow (工作流)
```go
type Workflow struct {
    ID          string
    Name        string
    Description string
    Version     string
    Status      WorkflowStatus  // DRAFT, PUBLISHED, ARCHIVED
    Nodes       []Node
    Edges       []Edge
    Variables   []Variable
    CreatedAt   time.Time
    UpdatedAt   time.Time
}
```

#### Node (节点)
```go
type Node struct {
    ID         string
    Type       NodeType    // START, END, LLM, PLUGIN, CODE, CONDITION, etc.
    Name       string
    Config     NodeConfig  // 节点配置（不同类型有不同结构）
    Position   Position    // 画布位置
    Inputs     []InputPin  // 输入引脚
    Outputs    []OutputPin // 输出引脚
}
```

#### Edge (边)
```go
type Edge struct {
    ID       string
    Source   string  // 源节点 ID
    Target   string  // 目标节点 ID
    SourcePin string // 源引脚
    TargetPin string // 目标引脚
    Condition string // 条件表达式（条件边）
}
```

```

│                    Workflow Execution                  │

                            │

┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
  1. 初始化
│  - 解析工作流定义
│  - 构建执行图

│      ├─ 创建检查点
│      └─ 触发后续节点
```

### 节点类型

| 节点类型 | 说明 |
|---------|------|
| **START** | 开始节点 |
| **END** | 结束节点 |
| **LLM** | 大模型调用节点 |
| **PLUGIN** | 插件调用节点 |
| **CODE** | 代码执行节点 (Python/JavaScript) |
| **CONDITION** | 条件判断节点 |
| **LOOP** | 循环节点 |
| **KNOWLEDGE** | 知识库检索节点 |
| **PROMPT** | 提示词模板节点 |
| **HTTP** | HTTP 请求节点 |
| **DATABASE** | 数据库查询节点 |

### 变量系统

工作流支持丰富的变量系统：
- **全局变量**：工作流级别的变量
- **节点输出变量**：每个节点的输出
- **环境变量**：系统环境变量
- **自定义变量**：用户定义的变量

变量引用语法：`${variable_name}` 或 `${node_id.output_name}`

智能体是 Coze Studio 的核心概念，负责与用户交互、调用工具、生成回复。

### 模块结构
```
domain/agent/singleagent/

│   ├── agent.go          # 智能体实体
│   ├── bot.go            # Bot 配置
│   └── prompt.go         # 提示词配置
├── repository/             # 仓储接口
│   └── agent_repo.go     # 智能体仓储
├── service/                # 领域服务
│   ├── agent_service.go  # 智能体服务
│   └── chat_service.go   # 聊天服务
└── internal/               # 内部实现
```

### 核心实体

#### Agent (智能体)
```go
type Agent struct {
    ID          string
    Name        string
    Description string
    Avatar      string
    Status      AgentStatus   // DRAFT, PUBLISHED, ARCHIVED
    Version     string
    
    // 核心配置
    BotConfig   BotConfig    // Bot 配置
    PromptConfig PromptConfig // 提示词配置
    
    // 关联资源
    WorkflowID  string       // 工作流 ID
    KnowledgeIDs []string     // 知识库 ID 列表
    PluginIDs   []string     // 插件 ID 列表
    
    CreatedAt   time.Time
    UpdatedAt   time.Time
}
```

#### BotConfig (Bot 配置)
```go
type BotConfig struct {
    ModelID       string        // 模型 ID
    Temperature   float64       // 温度参数
    TopP          float64       // Top-P 参数
    MaxTokens     int           // 最大 Token 数
    SystemPrompt  string        // 系统提示词
}
```

## 领域服务模式

每个领域模块都遵循统一的服务模式：

```go
// 仓储接口（定义在 Domain 层）
type XxxRepository interface {
    Create(ctx context.Context, entity *Xxx) error
    Update(ctx context.Context, entity *Xxx) error
    Delete(ctx context.Context, id string) error
    GetByID(ctx context.Context, id string) (*Xxx, error)
    List(ctx context.Context, filter *XxxFilter) ([]*Xxx, error)
}

// 领域服务
type XxxService struct {
    repo XxxRepository
    // 其他依赖...
}

func (s *XxxService) DoSomething(ctx context.Context, params *DoSomethingParams) (*Result, error) {
    // 1. 参数验证
    // 2. 业务逻辑
    // 3. 调用仓储
    // 4. 发布领域事件
    // 5. 返回结果
}
```

## 总结

Coze Studio 的 Domain 层采用了清晰的领域划分，每个领域模块都有完整的实体、仓储和服务。核心模块包括：

1. **workflow** - 工作流编排引擎
2. **agent** - 智能体核心
3. **knowledge** - 知识库 RAG
4. **plugin** - 插件系统
5. **conversation** - 对话管理
6. **prompt** - 提示词管理
7. **user** - 用户管理

每个模块都遵循 DDD 原则，通过接口进行依赖倒置，便于测试和扩展。

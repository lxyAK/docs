# Coze Studio 插件与扩展系统

## 插件系统概述

Coze Studio 的插件系统是其核心扩展能力之一，允许智能体调用外部工具、API 和服务。

## 插件架构

## 插件定义

### 插件实体结构

```go
type Plugin struct {
    ID          string
    Name        string
    Description string
    Icon        string
    Version     string
    Author      string
    Status      PluginStatus  // ACTIVE, INACTIVE, DEPRECATED
    
    // 工具列表
    Tools       []Tool
    
    // 认证配置
    AuthType    AuthType      // NONE, API_KEY, OAUTH2, BASIC
    AuthConfig  AuthConfig
    
    CreatedAt   time.Time
    UpdatedAt   time.Time
}
```

### 工具定义

```go
type Tool struct {
    ID          string
    Name        string
    Description string
    PluginID    string
    
    // 参数定义
    Inputs      []Parameter
    Outputs     []Parameter
    
    // 执行配置
    ExecuteType ExecuteType  // HTTP, RPC, CODE, BUILTIN
    ExecuteConfig ExecuteConfig
    
    // 元数据
    Categories  []string
    Tags        []string
}
```

### 参数定义

```go
type Parameter struct {
    Name        string
    Type        ParamType    // STRING, NUMBER, BOOLEAN, OBJECT, ARRAY
    Description string
    Required    bool
    Default     interface{}
    Enum        []interface{}  // 可选值列表
    Schema      string         // JSON Schema (复杂类型)
}
```

## 插件类型

### 1. HTTP 插件

通过 HTTP 请求调用外部 API。

```go
type HTTPExecuteConfig struct {
    URL         string
    Method      string  // GET, POST, PUT, DELETE, etc.
    Headers     map[string]string
    QueryParams map[string]string
    BodyTemplate string  // 请求体模板 (Go Template)
    ResponsePath string  // JSON Path (提取响应的特定部分)
    Timeout     int     // 超时时间 (秒)
}
```

### 2. RPC 插件

通过 gRPC 或 Thrift 调用服务。

### 3. 代码插件

在沙箱中执行代码（Python/JavaScript）。

```go
type CodeExecuteConfig struct {
    Language    string  // python, javascript
    Code        string  // 代码模板
    Timeout     int     // 超时时间 (秒)
    MemoryLimit int     // 内存限制 (MB)
    CPULimit    int     // CPU 限制 (核数)
}
```

### 4. 内置插件

Coze Studio 内置的工具函数。

## 插件认证方式

### 1. 无认证 (None)

公开 API，无需认证。

### 2. API Key

通过 Header 或 Query 参数传递 API Key。

```go
type APIKeyAuthConfig struct {
    KeyName    string  // Header 或 Query 参数名
    KeyValue   string  // API Key 值
    In         string  // header, query
}
```

### 3. Basic Auth

用户名/密码认证。

```go
type BasicAuthConfig struct {
    Username string
    Password string
}
```

### 4. OAuth2

OAuth2.0 认证。

```go
type OAuth2AuthConfig struct {
    ClientID     string
    ClientSecret string
    AuthURL      string
    TokenURL     string
    RedirectURL  string
    Scopes       []string
    AccessToken  string
    RefreshToken string
    ExpiresAt    time.Time
}
```

## 插件开发示例

### HTTP 插件定义示例 (YAML)

```yaml
id: weather
name: Weather
description: Get weather information
icon: 🌤️
version: 1.0.0
author: Coze

auth:
  type: api_key
  config:
    key_name: X-Api-Key
    in: header

tools:
  - id: get_current_weather
    name: Get Current Weather
    description: Get current weather for a city
    inputs:
      - name: city
        type: string
        description: City name
        required: true
      - name: unit
        type: string
        description: Temperature unit
        enum: [celsius, fahrenheit]
        default: celsius
    outputs:
      - name: temperature
        type: number
      - name: condition
        type: string
      - name: humidity
        type: number
    execute:
      type: http
      config:
        url: https://api.weatherapi.com/v1/current.json
        method: GET
        query_params:
          q: "{{ .city }}"
          aqi: "no"
        response_path: "$.current"
```

## 安全机制

### 1. 沙箱执行

代码插件在隔离的沙箱中执行。

### 2. 权限控制

插件调用需要用户授权。

### 3. 资源限制

- 超时限制
- 内存限制
- CPU 限制
- 网络限制

### 4. 审计日志

所有插件调用都有日志记录。

## 总结

Coze Studio 的插件系统设计灵活、安全、可扩展：

- **多种插件类型**：HTTP、RPC、代码、内置
- **完善的认证体系**：API Key、OAuth2、Basic Auth
- **清晰的扩展点**：自定义插件、节点、模型、存储
- **严格的安全机制**：沙箱、权限、资源限制、审计

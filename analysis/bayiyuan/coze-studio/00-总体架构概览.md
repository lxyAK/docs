# Coze Studio 总体架构概览

## 项目简介

Coze Studio 是源自字节跳动「扣子开发平台」的一站式 AI Agent 开发工具，现已完全开源。项目采用 **微服务架构 + 领域驱动设计 (DDD)** 原则构建，为开发者提供高性能、高扩展性的 AI Agent 开发环境。

**核心价值**：
- 提供 AI Agent 开发所需的全部核心技术（Prompt、RAG、Plugin、Workflow）
- 开箱即用，用最低的成本开发最专业的 AI Agent
- 可视化设计与编排工具，支持零代码/低代码开发

## 核心设计原则

### 1. 领域驱动设计 (DDD)
- **分层清晰**：API → Application → Domain → Infrastructure
- **依赖倒置**：Domain 层不依赖 Infrastructure，通过接口抽象
- **领域模型**：核心业务逻辑封装在 Domain 层

### 2. 微服务架构
- 按领域划分服务边界
- 通过 IDL 定义服务接口
- 支持独立部署和扩展

### 3. 可扩展性
- 插件化设计
- 模块化架构
- 清晰的扩展点

## 启动流程

### 后端启动 (main.go)

```go
func main() {
    ctx := context.Background()
    
    // 1. 设置崩溃输出
    setCrashOutput()
    
    // 2. 加载环境变量
    if err := loadEnv(); err != nil {
        panic("loadEnv failed")
    }
    
    // 3. 设置日志级别
    setLogLevel()
    
    // 4. 初始化基础设施 (DB, Cache, 外部服务等)
    if err := application.Init(ctx); err != nil {
        panic("InitializeInfra failed")
    }
    
    // 5. 启动 HTTP 服务器
    startHttpServer()
}
```

### HTTP 服务启动

```go
func startHttpServer() {
    // 创建 Hertz 服务器
    s := server.Default(opts...)
    
    // 中间件链 (顺序很重要!)
    s.Use(middleware.ContextCacheMW())     // 1. 上下文缓存
    s.Use(middleware.RequestInspectorMW()) // 2. 请求检查
    s.Use(middleware.SetHostMW())           // 3. 设置 Host
    s.Use(middleware.SetLogIDMW())          // 4. 设置 Log ID
    s.Use(corsHandler)                       // 5. CORS
    s.Use(middleware.AccessLogMW())          // 6. 访问日志
    s.Use(middleware.OpenapiAuthMW())        // 7. OpenAPI 认证
    s.Use(middleware.SessionAuthMW())        // 8. 会话认证
    s.Use(middleware.I18nMW())               // 9. 国际化
    
    // 注册路由
    router.GeneratedRegister(s)
    
    // 启动服务
    s.Spin()
}
```

## 关键中间件说明

| 中间件 | 位置 | 作用 |
|--------|------|------|
| **ContextCacheMW** | 1st | 上下文缓存，必须第一个 |
| **RequestInspectorMW** | 2nd | 请求检查，必须第二个 |
| **SetHostMW** | 3rd | 设置 Host 信息 |
| **SetLogIDMW** | 4th | 设置日志追踪 ID |
| **CORS** | 5th | 跨域处理 |
| **AccessLogMW** | 6th | 访问日志记录 |
| **OpenapiAuthMW** | 7th | OpenAPI 鉴权 |
| **SessionAuthMW** | 8th | 会话鉴权 |
| **I18nMW** | 9th | 国际化 (必须在 SessionAuthMW 之后) |

## 总结

Coze Studio 采用了经典的 **DDD 分层架构**，结合字节跳动内部的开源技术栈（Hertz、Eino、FlowGram），构建了一个高性能、可扩展的 AI Agent 开发平台。

**架构亮点**：
1. 清晰的分层，职责明确
2. 领域驱动设计，核心业务逻辑集中
3. 依赖倒置，易于测试和扩展
4. 完善的中间件链，处理横切关注点
5. 基于字节跳动开源技术栈，可靠性高

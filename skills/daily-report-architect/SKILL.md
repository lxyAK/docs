# 技能：智能架构师日报生成器
## 技能描述
结合Git Commit记录和大模型智能分析能力，自动生成高质量架构师视角的日报。核心是**按业务功能分类**、**业务聚合**、**价值提炼**和**智能分类**，避免模板化描述。
## 使用场景
- 每日定时自动生成架构师视角的工作日报
- 将跨项目、零散的Git提交记录转化为结构化、有业务价值的汇报材料
- 支持人工补充手动完成的工作（如安全扫描、文档编写等）
## 核心能力
### 按业务功能分类（核心原则）
**不再按仓库名称分组**，而是完全根据实际提交内容对应的业务功能进行智能分类和命名**，合并为一个"业务功能"条目汇报，体现全链路架构视野。

例如：
- ❌ 错误：按"weComH5"、"hospitalInPc"等仓库分组
- ✅ 正确：按"医疗报告功能体验优化"、"商城模块规划建设"等业务功能分组

### 价值翻译与噪音过滤
- **严禁**出现"完成功能迭代与优化"等无意义描述
- **自动忽略**纯`chore`类的依赖更新或锁文件变动，除非涉及重大安全修复
- **价值翻译规则**：
  - `perf`类提交 → 翻译为"性能提升/资源节省"
  - `feat`类提交 → 翻译为"业务赋能/合规支持"
  - `fix`类提交 → 翻译为"稳定性提升/故障规避"
  - `docs`类提交 → 翻译为"技术沉淀/规范对齐"
### 智能洞察
- 自动识别代码变动中的潜在风险（如数据库变更、并发策略调整、依赖升级等）并给出应对建议
- 自动统计性能优化、问题修复、文档沉淀等架构治理类工作成果
### 人工补充支持
预留接口让用户补充手动完成的工作内容（如：安全扫描、会议、调研等）
## 执行标准流程
### 第一步：自动采集数据
运行`scripts/daily_report_sender.py`自动拉取所有项目当日全量提交记录作为基础数据（备注：`arkclaw-lxy01/daily_report_sender.py`是备份版本）
### 第二步：大模型智能分析
1. 分析实际提交内容，**按业务功能智能分类**（拒绝按仓库分组）
2. 提取有价值的技术点，根据实际commit message生成精准描述
3. 自动统计架构治理数据（性能优化、问题修复、文档沉淀等）
4. 识别潜在风险点
### 第三步：人工补充内容
用户可以补充手动完成的工作内容（如：安全扫描、会议、调研、文档编写等）
### 第四步：生成标准格式日报
严格按照2026-03-31的标准格式生成日报，确保：
- **按业务功能分类**，不按仓库分组
- 业务分类准确，符合实际工作内容
- 技术点描述具体，不使用模板化语言
- 业务价值提炼精准
- 架构治理数据准确
- 风险识别到位
### 第五步：发送与归档
1. 发送给用户确认
2. 按日期归档到指定目录
## 日报输出标准模板
严格遵循以下格式（参考2026-03-31日报）：
```markdown
# 【架构师日报】YYYY-MM-DD
**【今日核心产出】**
- **[业务功能名称]**（涉及：`项目A`, `项目B`）
    - **业务价值**：[一句话总结核心价值，如：提升医疗报告页面加载稳定性与代码可维护性，降低用户等待时间]
    - **关键工作**：
        - [前端/交互]：[根据实际提交提炼，如：修复小结报告首屏loading卡住问题，页面加载成功率提升至100%]
        - [后端/核心]：[根据实际提交提炼，如：实现Puppeteer浏览器实例复用，降低内存消耗]
- **[业务功能名称]**（涉及：`项目C`）
    - **业务价值**：...
    - **关键工作**：...
- **[手动补充的工作]**（涉及：安全保障/会议/调研等）
    - **业务价值**：...
    - **关键工作**：...
**【架构与质量治理】**
- **质量优化**：完成X项前端问题修复，核心功能稳定性提升
- **功能迭代**：完成X项新功能开发，业务能力持续增强
- **安全保障**：完成X项安全专项工作，系统安全性提升
- **文档沉淀**：完成X项技术文档输出，架构资产持续积累
**【风险与明日规划】**
- **潜在风险**：[根据代码变动和工作内容推测，如：供应链风险排查发现5个无法扫描的仓库，需确认仓库状态]
- **明日重点**：[基于今日进度，规划后续验证或开发工作，如：完成今日迭代功能的回归验证，持续推进核心功能开发]

---
📅 生成时间: YYYY-MM-DD HH:MM:SS
```
## 业务分类原则
**完全以实际提交内容为核心，智能分析判断**：
1. **按业务功能分类**，不按仓库名称分组
2. 根据提交内容对应的实际业务模块智能分类命名，不局限于预设分类
3. 相同业务领域的跨项目提交合并归类，体现全链路工作成果
4. 业务价值描述完全根据实际工作产出总结，精准对应业务场景
5. 技术点描述根据实际commit message生成，拒绝模板化
## 使用说明
### 触发方式
当用户说"生成今天的日报"、"写日报"、"今日工作总结"时，自动执行此技能。
### 执行步骤
1. 先运行数据采集脚本获取基础数据
2. 分析提交记录，**按业务功能智能分类**和提炼
3. 生成标准格式日报
4. 发送给用户确认
5. 归档保存
